# 图谱分组抽屉视觉统一设计

日期：2026-06-30
范围：llm-wiki-agent 图谱右侧社区抽屉与选区抽屉
视觉定稿：`designs/community-drawer-visual-options/index.html`，以 `preview-selected-centered.png` 中的「B 方案：更克制主色版 / 3 更克制」为准

## 背景

当前统一社区抽屉已经把社区、未分组、Shift 多选和邻居选区收敛到同一类工作流，但视觉还存在几处明显不统一：

1. 顶部状态标签放在统计区之后，信息层级不自然。
2. “结构清晰”等状态没有和“社区 / 选区”形成同一组元信息。
3. 统计数字上下两行显示，占用高度，也让数字和标签显得割裂。
4. 推荐动作的浅色胶囊曾出现偏粉的问题，不耐看，也和项目主色不够一致。
5. 社区抽屉和选区抽屉如果只改一边，后续会再次形成两套视觉标准。

本次设计将用户确认的第 3 版视觉稿定为图谱分组抽屉的统一视觉标准。实现时只允许调整视觉和必要布局，不改变任何业务功能。

## 设计目标

1. 社区抽屉和选区抽屉共用同一套视觉标准。
2. 严格复现定稿视觉稿的核心观感：暖纸面、克制主色、轻胶囊、单行统计、清楚但不刺眼的主操作。
3. “社区 / 选区”和“结构清晰”等状态标签移动到顶部并列显示。
4. 统计数字和文字在胶囊内横向排列并严格居中，不再分成上下两行。
5. 保留所有已有动作和对话能力，不改 prompt、选区、进入社区、发送、新对话等行为。

## 非目标

1. 不重做图谱主画布。
2. 不改单节点摘要、搜索结果摘要、全局概览、错误态和加载态。
3. 不新增图谱功能、对话功能或社区阅读功能。
4. 不新增主题色体系，不引入粉色、紫色或其他独立配色。
5. 不改变动作推荐逻辑，也不改变按钮点击后的结果。

## 视觉基准

### 色彩

所有颜色必须来自项目现有主题变量，不再凭截图猜色。

主色体系：

- 页面背景：`--app-bg`
- 卡片表面：`--app-surface`
- 抬起底色：`--app-raised`
- 主文字：`--app-fg`
- 次级文字：`--app-muted`
- 边框：`--app-border`
- 强边框：`--app-border-strong`
- 主色：`--app-accent`
- 主色深色：`--app-accent-deep`
- 主色浅底：`--app-accent-soft`
- 成功提示：`--app-success`

视觉规则：

- 只有“发送”按钮使用实心主色。
- “进入社区”、顶部标签、统计项、推荐动作只使用浅底、纸面底和细描边。
- 推荐动作可以带主色倾向，但不能变成粉色块，也不能比“发送”更抢眼。
- “新对话”保持中性纸面按钮，不使用实心主色。

### 形状与质感

抽屉整体延续纸面风格：

- 外层卡片为暖色纸面底，保留轻微纹理感和柔和阴影。
- 内部信息区用细边框和浅底分层，不使用大面积强色块。
- 标签、统计项、动作按钮使用胶囊形态。
- 节点列表和输入框继续使用较小圆角，避免整个界面过度圆润。

## 布局设计

### 顶部信息区

顶部信息区是视觉落地的重点。

结构顺序：

1. 元信息胶囊行。
2. 标题。
3. “进入社区”按钮。
4. 描述文字。

元信息胶囊行：

- 社区抽屉显示“社区”和结构状态，例如“结构清晰”。
- 选区抽屉显示“选区”和选区状态，例如“Shift+点击增删节点”。
- 多个标签在同一行从左到右排列，可以换行，但不能被放到统计数字后面。
- 状态标签带一个小圆点，使用主色弱提示。

标题与按钮：

- 标题保持主要视觉重量。
- “进入社区”放在标题区域右侧，使用轻量胶囊按钮。
- 没有进入社区能力时，不显示该按钮，也不占位。

描述文字：

- 描述位于标题下方。
- 文案保持现有内容，不改含义。
- 行高要舒展，但不能撑高整个首屏。

### 统计条

统计条改为单行四列胶囊。

要求：

- 四个统计项保持同一行：页、链接、核心 / 社区、孤立。
- 每个统计项是等宽胶囊。
- 胶囊内数字和标签横向排列。
- 数字和标签作为整体在胶囊内水平、垂直居中。
- 数字使用主色深色，标签使用次级文字色。
- 不能再出现数字在上、标签在下的两行统计。
- 不能用基线对齐造成数字偏上或偏下。

### 动作区

动作区继续固定四个动作：

1. 总结这一簇
2. 找知识缺口
3. 生成主题页
4. 探索潜在关系

视觉要求：

- 两列网格排列。
- 所有动作是轻量胶囊按钮。
- 推荐动作使用主色浅底和弱主色边框。
- 推荐动作不使用实心主色，避免和“发送”抢主操作。
- 普通动作使用中性纸面底。

行为保持不变：

- 社区和选区仍按现有推荐逻辑标记推荐动作。
- 点击动作仍调用现有提问流程。

### 节点区

节点区继续展示核心节点或选区节点。

视觉要求：

- 区块保持轻卡片样式。
- 标题右侧的“查看全部 / 收起”继续保留。
- 节点行保持可点击感：轻边框、浅底、悬停 / 聚焦反馈。
- 节点标题过长时省略，不撑破布局。

行为保持不变：

- 社区核心节点超过 3 个时仍可展开。
- 选区抽屉不新增展开能力。
- 点击节点仍打开节点摘要。
- 悬停和聚焦仍触发图谱预览。

### 对话区

对话区保持现有能力，但视觉跟随定稿。

要求：

- 输入框继续显示“补充说明（可选）”。
- 上下文提示保留绿色小点：
  - 社区抽屉：`当前社区会带入对话`
  - 选区抽屉：`当前选区会带入对话`
- 底部按钮两列等宽。
- “发送”在左，使用实心主色。
- “新对话”在右，使用中性纸面按钮。
- 图标保留，按钮文字和图标居中。

行为保持不变：

- 空输入时“发送”仍禁用。
- 输入内容后“发送”仍启用。
- “新对话”仍走现有新对话逻辑。

## 社区与选区统一规则

社区抽屉和选区抽屉使用同一个抽屉组件和同一套样式。

允许差异：

- 顶部类型胶囊文案不同：社区 / 选区。
- 结构或状态标签来自各自已有 view model。
- 社区抽屉可显示“进入社区”。
- 选区抽屉不显示“进入社区”。
- 社区节点列表可展开。
- 选区节点列表保持当前规则。

不允许差异：

- 统计条不能一边单行、一边两行。
- 推荐动作不能一边浅底、一边实心色。
- 对话按钮不能一边有图标、一边无图标。
- 顶部状态标签不能一边上移、一边仍留在统计后面。

## 工程边界

实现应集中在现有统一抽屉相关文件：

- `workbench/web/src/components/GraphGroupDrawer.tsx`
- `workbench/web/src/index.css`
- 必要时补充相关前端测试

原则：

- 不改变数据来源。
- 不改变动作回调。
- 不改变 prompt 拼接。
- 不改变图谱引擎。
- 不改变社区阅读视图。
- 不新增依赖。

若实现顶部标签上移需要调整 DOM 结构，只能在现有 `GraphGroupDrawer` 内做轻量重排。标签数据仍使用现有 `view.tags`。

## 验收标准

实现完成后必须同时检查社区抽屉和选区抽屉。

社区抽屉验收：

1. 顶部“社区”和“结构清晰”等标签并列显示。
2. “进入社区”在顶部信息区右侧，样式为轻量胶囊。
3. 统计项一行显示，数字和标签在胶囊内居中。
4. 推荐动作不是粉色块，也不是实心主色。
5. “发送”是唯一实心主色按钮。
6. 节点展开、节点点击、输入、发送、新对话功能都正常。

选区抽屉验收：

1. 顶部“选区”和选区状态标签并列显示。
2. 没有“进入社区”按钮。
3. 统计项一行显示，数字和标签在胶囊内居中。
4. 动作区、节点区、对话区视觉与社区抽屉一致。
5. 输入、发送、新对话功能都正常。

视觉对照验收：

1. 实际页面要接近 `preview-selected-centered.png` 的整体观感。
2. 统计数字不能偏上、偏下或只靠基线对齐。
3. 胶囊背景不能出现用户已否定的粉红感。
4. 抽屉首屏信息密度要比当前线上版本更紧凑。
5. 文案不能挤压、重叠或溢出按钮。

## 测试要求

需要运行相关前端检查，并进行真实浏览器验证。

自动检查：

- 前端相关测试应覆盖社区和选区抽屉仍渲染关键内容。
- 样式契约需要保护：
  - 统计条使用单行布局。
  - `graph-selection-send` 仍使用 `var(--app-accent)`。
  - `graph-selection-secondary` 仍使用 `var(--app-raised)`。
  - 推荐动作仍通过 `data-recommended="true"` 呈现浅主色样式。

浏览器检查：

1. 打开图谱，点击普通社区。
2. 检查顶部标签、统计条、推荐动作和底部按钮。
3. 输入补充说明，确认发送按钮状态变化。
4. 点击 Shift 多选或邻居选区，检查选区抽屉采用同一标准。
5. 检查统计胶囊在实际抽屉宽度下仍保持居中。

## 完成定义

本次实现只有在以下条件同时满足时才算完成：

1. 社区和选区抽屉都复现定稿视觉。
2. 用户指出的两个问题都被消除：
   - “进入社区”视觉不再突兀。
   - 统计数字不再占两行，且在胶囊内居中。
3. 新发现的胶囊配色问题被消除，不再出现不耐看的粉红感。
4. “结构清晰”等状态已移动到顶部并和“社区 / 选区”同组。
5. 所有原有功能保持可用。
6. 浏览器中实际打开页面验证通过。
