# PR82 抽屉恢复与视觉收敛设计

日期：2026-06-30  
范围：llm-wiki-agent 图谱右侧统一社区/选区抽屉  
相关 PR：#82 `v3.6.16 fix: restore graph community interaction flow`  
视觉参考：`designs/pr82-drawer-redesign/index.html`

## 背景

PR82 合并后，统一社区抽屉解决了普通社区、未分组、Shift 多选和邻居选区的心智割裂问题，但也带来两类回退：

1. 交互能力回退：
   - 核心节点统计显示超过 3 个时，列表仍只露出 3 个，没有“查看全部 / 收起”。
   - 核心节点行可点击、可预览，但鼠标移上去缺少明显反馈。
   - 对话区缺少轻提示，用户不知道当前社区或选区会被带入对话。

2. 视觉层级跑偏：
   - 抽屉按钮颜色和边框偏重，容易把抽屉看成操作面板。
   - 发送按钮需要保留旧版清晰的主色和纸飞机图标，但整体仍要融入新版米色纸张风格。

本次设计只做 PR82 后续修复：找回遗失能力，并收敛抽屉视觉。不得扩大成新功能重构。

## 已确认边界

### 本次要做

1. 社区核心节点超过 3 个时，显示“查看全部 / 收起”。
2. 核心节点行恢复清楚的鼠标反馈，同时继续支持点击查看节点摘要。
3. 对话区显示轻提示：
   - 社区抽屉：`当前社区会带入对话`
   - 选区抽屉：`当前选区会带入对话`
4. 对话区视觉按最终稿收敛：
   - “发送”按钮采用主色、纸飞机图标。
   - “新对话”按钮采用次级样式、图标。
   - 两个按钮等宽。
   - 输入框获得焦点时沿用主色边框反馈。

### 本次不要做

1. 不改单个知识点 / 页面详情抽屉。
2. 不改全局搜索弹窗。
3. 不改社区阅读页。
4. 不改图谱布局、节点位置、颜色、连线算法。
5. 不把搜索命中明细、固定节点明细、桥接关系列表放回抽屉首屏。
6. 不新增“选中页面查看全部”。选区抽屉本次只恢复对话提示和按钮视觉，不扩展页面列表能力。

## 产品设计

### 抽屉结构

继续沿用 PR82 的统一抽屉结构：

1. 顶部概览：类型、标题、说明、统计、进入社区按钮。
2. 下一步动作：总结这一簇、找知识缺口、生成主题页、探索潜在关系。
3. 核心节点：社区抽屉默认展示前 3 个核心节点，超过 3 个时可展开。
4. 对话区：补充说明、轻提示、发送、新对话。

这四块保持稳定，不新增搜索、固定、桥接等大块信息。

### 核心节点展开

社区抽屉的核心节点区默认显示 3 个节点。

当核心节点总数大于 3 时：

- 区块标题右侧显示“查看全部”。
- 点击后展示全部核心节点，并把按钮文案切为“收起”。
- 再次点击后恢复只显示前 3 个。
- 统计里的“核心”仍显示核心节点总数，不显示当前露出的数量。

没有核心节点时继续显示空状态，不影响动作区和对话区。

### 核心节点鼠标反馈

核心节点行保持现有行为：

- 鼠标移入 / 聚焦：触发图谱预览。
- 鼠标移出 / 失焦：取消预览。
- 点击：切换到节点摘要。

视觉上需要补回反馈：

- 行背景轻微变亮或带主色浅底。
- 边框变成主色弱提示。
- 可有轻微位移或内侧主色提示线。
- 反馈必须同样覆盖键盘聚焦态。

### 对话轻提示

对话区输入框和按钮之间显示轻提示。

社区抽屉显示：`当前社区会带入对话`。未分组虽然是虚拟社区，也使用这句提示。  
选区抽屉显示：`当前选区会带入对话`。

提示只解释当前上下文会进入对话，不展示页面清单，不展开 prompt 内容。

### 对话按钮视觉

对话区采用最终设计稿的按钮逻辑：

- “发送”在左，“新对话”在右。
- 两个按钮等宽。
- “发送”启用时使用 `var(--app-accent)` 主色和 `Send` 图标。
- “新对话”使用次级按钮样式和 `MessageSquarePlus` 图标。
- 禁用态仍要明显不可点击，不要误导用户。
- 两个按钮都保留 hover/focus 反馈。

发送行为不变：

- 当前对话发送继续调用现有 `onAsk(null)`。
- 新对话继续调用现有 `onAskInNewConversation(null)`。
- 空输入的新对话仍沿用现有推荐动作逻辑。

## 工程设计

### 组件边界

改动集中在：

- `workbench/web/src/lib/graph-group-drawer.ts`
- `workbench/web/src/components/GraphGroupDrawer.tsx`
- `workbench/web/src/index.css`
- 相关前端测试

不修改图谱引擎的数据来源、选区生成、prompt 通道、社区阅读视图。

### View model

当前 `graphCommunityDrawerViewModel()` 把 `payload.coreNodes` 直接截断为 3 个，导致组件无法知道还有多少节点可展开。

本次应让社区抽屉拿到完整核心节点列表，由组件控制默认展示和展开状态。

要求：

- 社区抽屉保留完整 `nodes`。
- 选区抽屉维持当前最多 3 个选中页面，不新增展开。
- 统计中的核心数量继续来自 `payload.coreNodeIds.length`。

view model 需要增加两个简单字段，帮助组件区分是否允许展开，以及切换对象时如何重置展开状态：

- `nodeListExpandable: boolean`
- 社区为 `true`
- 选区为 `false`
- `nodeListKey: string`
- 社区使用稳定社区 id，例如 `community:${payload.communityId}`
- 选区使用稳定 selection id，例如 `selection:${selection.id}`

### 组件状态

`GraphGroupDrawer` 内部维护一个轻量展开状态。

要求：

- 切换到新的社区或选区时，展开状态自动回到收起。
- 展开状态以 `view.nodeListKey` 为重置依据，不用标题文本做依据，避免同名社区或同名选区造成状态串台。
- 默认展示前 3 个节点。
- 只有 `nodeListExpandable === true` 且节点数大于 3 时显示“查看全部 / 收起”。

### 样式

样式继续使用现有主题变量：

- 主色：`--app-accent`
- 主色深色：`--app-accent-deep`
- 弱主色底：`--app-accent-soft`
- 边框：`--app-border`
- 抬起背景：`--app-raised`

不要引入新配色体系。

## 必须保护的现有能力

1. 普通社区和未分组仍走同一个统一抽屉。
2. 未分组不显示“进入社区”。
3. “探索潜在关系”仍对所有社区可见，并在未分组时作为推荐动作。
4. Shift 多选仍打开统一选区抽屉。
5. 单节点“+邻居”仍能扩展为邻居选区。
6. 进入社区仍能进入社区阅读视图。
7. 点击核心节点仍打开节点摘要。
8. 搜索命中为空、固定节点为空时，不出现独立大块空状态。

## 测试要求

### 单元 / DOM 测试

需要覆盖：

1. 社区 view model 保留超过 3 个核心节点，统计显示总数。
2. 社区抽屉默认只渲染前 3 个核心节点。
3. 点击“查看全部”后显示全部核心节点，再点“收起”恢复 3 个。
4. 选区抽屉不显示“查看全部 / 收起”。
5. 社区抽屉显示 `当前社区会带入对话`。
6. 选区抽屉显示 `当前选区会带入对话`。
7. 发送和新对话按钮都带图标，并使用同一两列网格样式保证等宽。
8. 点击核心节点仍调用节点摘要回调。

### 浏览器验证

需要实际打开应用检查：

1. 点击普通社区：核心节点超过 3 个时可展开 / 收起。
2. 鼠标移到核心节点行：行有明显反馈，图谱节点预览仍可见。
3. 输入补充说明：发送按钮进入可发送状态，颜色采用主色。
4. 点击未分组：不出现“进入社区”，对话提示显示 `当前社区会带入对话`。
5. Shift 多选：仍能打开统一选区抽屉，对话提示存在。
6. 单节点“+邻居”：仍能进入统一选区抽屉。
7. 发送和新对话按钮实际渲染宽度一致。

## 验收标准

1. 用户能看到核心节点总数和完整列表，不再出现“显示 5 个核心但只能看 3 个”的问题。
2. 核心节点行看起来可点击、可悬停，不再像静态文本。
3. 用户知道当前社区或选区会被带入对话。
4. 发送按钮视觉回到明确主操作，同时不让整个抽屉显得过重。
5. PR82 已恢复的社区/未分组/多选/邻居选区能力不回退。
6. 抽屉首屏仍保持克制，不重新塞入搜索、固定、桥接等复杂信息。
