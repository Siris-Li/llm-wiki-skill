## 1. 问题与事实

`packages/graph-engine/src/model/legacy-helpers.ts` 仍是图谱引擎的真实实现中心。它有 1414 行、45 个导出，并通过 `@ts-nocheck` 跳过严格类型检查。文字处理、搜索、社区与旧数据兼容、图谱模型归一化、初始布局、可见范围、缩放、小地图、存储和早期阅读队列等不同职责都混在这一个文件中。

仓库已经存在 `labels.ts`、`visibility.ts`、`atlas.ts`、`storage.ts`、`queue.ts` 和 `learning.ts` 等分类入口，但它们大多只把函数继续转发到旧文件。分类名称已经有了，真实职责仍未拆开；`render/search.ts` 甚至还直接引用旧文件。

这不是 DOM/SVG 社区图独有的遗留问题。当前两条绘制路线都经过同一份共享模型：

```text
原始图谱数据
    |
    v
共享模型、布局、搜索与可见性规则
    |
    +--> Sigma 主路线
    |
    +--> DOM/SVG 故障回退
```

工作台与离线单文件图谱也都通过 `createGraphEngine` 使用同一个引擎。因此，旧工具箱仍处在 Sigma、故障回退和两个宿主的共同上游。#159 创建于 Sigma 社区阅读主路线完成之后，并未被 Sigma 迁移自然解决。

当前基线已经实际验证：

- 图谱引擎 761 项测试全部通过。
- 图谱引擎严格类型检查通过，但旧工具箱本身被整文件跳过。
- 工作树在设计开始时干净。
- 远端 #159 仍为开放状态，无子 issue、无评论、无阻塞项。

## 已有基础（What already exists）

本次不从零重建图谱引擎，直接复用以下现有基础：

- `model/labels.ts`、`model/visibility.ts`、`model/atlas.ts` 等分类入口已经存在，本次把真实实现和类型放回这些负责人，不再加新的总入口。

- `render/viewport.ts` 已承担当前相机、投影、缩放、平移和小地图换算，只收回仍向旧工具箱借用的规则。
- `render/model.ts` 与 `render/adapter.ts` 已经是 Sigma 和 DOM/SVG 共用规则的现行通路，本次缩小其职责，不另建第二条绘制管线。
- `select/`、`summary/` 和 facade 已有独立选择、摘要与宿主状态语义，本次不迁入共享绘制模型。
- `architecture.ts`、现有边界测试、761 项图谱引擎测试、大图 fixture、工作台与离线浏览器回归都继续使用，并按新的真实边界补强。
- ESM 与 IIFE 双产物、工作台加载和离线单文件生成已经落地，本次验证它们继续消费同一引擎，不改变分发方式。

## 2. 目标

本次工作只改变内部职责，不改变任何用户可见行为。

完成后必须达到以下结果：

1. 旧工具箱文件彻底删除，不保留过渡壳。
2. 图谱引擎源码不再依赖 `@ts-nocheck` 维持运行。
3. 文字、模型、搜索与可见性、布局、视角控制各有唯一且清楚的负责人。
4. 共享模型提供明确输入输出类型，下游不再用重复的临时类型和强制转换猜测结果。
5. Sigma、DOM/SVG 故障回退、工作台和离线单文件图谱继续得到相同语义。
6. 确认无人使用的旧能力、旧出口和早期离线模板一并删除。
7. 长期边界检查阻止旧工具箱、直接绕行和跳过类型检查重新出现。

- 不改变图谱外观、交互、文案、动效或性能目标。

- 不顺手重构与共享模型无关的工作台、服务端或图谱数据生成流程。
- 不为了追求新结构而停止兼容现有旧图谱数据。

## 4. 三类“旧代码”的边界

仓库里有三类容易混淆的旧代码，必须分别处理。

### 4.1 共享模型旧工具箱

这是 #159 的核心对象。Sigma 和 DOM/SVG 都仍在使用其中的规则。它必须被分批迁移并最终删除。

### 4.2 DOM/SVG 故障回退

这是当前引擎的一条真实运行路线，只在 Sigma 初始化或运行失败时启用。它继续消费共享模型，但本轮不改变其产品地位、不新增功能、不删除。

### 4.3 早期离线 wash 模板

`templates/graph-styles/wash/` 是引擎迁移前的旧离线实现。当前离线 HTML 已改用图谱引擎 IIFE，生成结果也明确不再加载 `graph-wash.js` 和 `graph-wash-helpers.js`。旧设计要求它只保留一个版本周期，相关测试也已经标明应在同一退休周期删除。

本次最终收尾包含该目录及其专属测试的删除。删除前必须确认其中没有仍被当前引擎使用的主题、样式或数据契约；若发现仍有效的资产，先迁到当前负责人，再删除旧模板。生成脚本不再复制或加载旧文件，但必须继续删除已有知识库目录中由旧版本遗留的 `graph-wash.js` 与 `graph-wash-helpers.js`，避免升级后残留废弃资源。

“wash” 名称本身不是删除依据。当前引擎的 `render/community-wash.ts`、`render/community-washes.ts`、`.community-wash` 视觉和 `graph-community-wash-interactions` 浏览器回归仍属于现行 Sigma 能力，必须保留。专属旧模板测试以文件内 `DEPRECATED` 标记和直接导入 `templates/graph-styles/wash/` 为准。

## 5. 目标架构

Sigma 与 DOM/SVG 故障回退共用的绘制数据流保持单向：

```text
原始 GraphData
    |
    v
归一化图谱模型
    |
    +--> 初始世界布局与固有范围
    |        |
    |        v
    |    应用 Pin 与实时拖动位置
    |        |
    |        v
    |    最终位置与内容范围
    |
    +--> 搜索、筛选与语义可见集合
             |
             v
       视口感知的绘制策略
       （取景范围、密度与预算）
             |
             v
       可绘制规则快照组装
             |
             +--> Sigma 适配
             |
             +--> DOM/SVG 回退适配

内容范围 + 取景范围 --> 视角控制 --> 屏幕位置与小地图
```

这张数据流图只描述从原始输入到可绘制快照的共享绘制链路，不代表整个引擎必须改用一个全局模型。引擎入口先把 `unknown` 运行时输入投影成结构安全的兼容 `GraphData`，同时保留常规搜索需要的原始兼容信息；facade 持有这份兼容数据。`select/` 与 `summary/` 继续按各自语义建立小型索引，本次不迁移或合并它们，也不让归一化图谱模型接管选择、摘要或宿主状态。

图谱含义、初始世界位置、最终内容位置和屏幕视角是不同层次：

- 图谱含义回答“节点、边和社区是什么”。
- 初始世界位置和固有范围回答“没有用户位置覆盖时，它们在同一张地图上的位置和自然边界是什么”。
- 最终内容位置和内容范围回答“应用 Pin 与当前拖动位置后，节点实际上在哪里、地图至少要包住哪里”。
- 取景范围回答“为了适配当前屏幕比例，需要在自然边界外补多少空间”。
- 屏幕视角回答“用户此刻看到取景范围的哪一部分”。

后一个层次可以依赖前一个层次的结果，但不能反向修改前一个层次的事实。

当前社区阅读会根据 `viewportSize` 扩展 `worldBounds` 以锁定屏幕宽高比，并根据屏幕宽度调整标签预算。这是现有行为，必须保留。目标架构将它明确为“视口感知的绘制策略”，而不是初始布局：

- `layoutBounds` 只由初始世界位置产生，不读取屏幕尺寸。
- 最终位置的优先级保持为“实时拖动位置 > 持久化 Pin > 初始布局位置”。
- `contentBounds` 保持当前输入集合：所有通过类型筛选的节点，加上临时显示节点；社区聚焦不能在本轮把它缩成仅当前社区。它按这些节点的最终位置和现有留白规则产生，必须包住位于初始范围外的 Pin 或拖动节点。
- `framingBounds` 基于 `contentBounds` 和当前视口比例扩展，用于社区阅读取景。
- 标签预算可以读取视口分类，但不能改变初始世界位置和结构事实。

## 6. 模块职责

### 6.1 文字与标签

负责：

- 按字素安全拆分中文、拉丁字符、组合字符和 emoji。
- 标签宽度估算、长标题截断和卡片尺寸。
- 摘要中的 Markdown 清理。
- 节点类型、关系置信度等现有显示文案。

该模块只处理文字和显示所需的纯计算，不读取图谱状态、不决定节点可见性、不操作 DOM。

### 6.2 归一化图谱模型

运行时边界先一次性生成 `GraphInputProjection`：

- `data` 是结构安全的兼容 `GraphData`，至少保证顶层对象、节点数组、边数组和节点 ID 可被 facade、选择、摘要与路线判断安全读取。
- `regularSearchByNode` 保留字段修剪与转换前的常规搜索兼容投影。

这一步只保证数据形状安全，不接管选择、摘要或宿主语义。随后，绘制链路把兼容 `GraphData` 整理成稳定、带类型的内部模型，包括：

- 节点、边、社区、起始节点和图谱洞察。
- 旧字段、缺失字段、未知值与默认值兼容。
- 节点、边和社区的结构索引，例如 `byId` 与 `communityById`。
- 携带运行时投影阶段已经生成的常规搜索兼容信息，固定旧标题真假值、ID 回退和正文前 500 个 UTF-16 代码单元；绘制归一化不得再次生成。
- 仍影响当前图谱结果的旧 learning 数据兼容。

这里的“共享”是指 Sigma 与 DOM/SVG 故障回退共享，不是全引擎唯一数据存储。该模块输出绘制链路共用的节点、边、社区、布局输入和可见快照类型。`render/model.ts` 不再维护另一套重复的 Atlas 临时类型，也不再通过 `as` 强行声明旧工具箱的返回形状。

运行时投影与绘制归一化都是总函数：接收 `unknown` 或字段不完整的运行时值时不得抛错，始终返回结构完整、顺序确定的结果；非法集合按现有宽容规则变为空集合，非法对象按字段级默认值或过滤规则处理。类型收紧不能把原先可容忍的数据错误变成同步异常。

本轮保留现有 ID 碰撞语义：节点、边和 learning 社区数组保持原顺序与重复项；按 ID 建立的索引采用最后写入项；缺失节点或边 ID 继续按数组下标生成，且不额外避让同名真实 ID；关系端点过滤、度数、社区成员和起点结果继续按迁移前规则计算。更合理的去重、拒绝或重命名策略属于后续行为变更。

每次新数据进入引擎只生成一次运行时投影；一次绘制更新只生成一次绘制模型，结果由布局、可见性和绘制策略继续消费。下游不能为了方便重新从原始输入或兼容 `GraphData` 建另一份等价模型。

旧 learning 的兼容规则若只被模型归一化内部使用，就保留为私有实现，不继续作为独立公共工具箱出口。

### 6.3 搜索与可见范围

负责：

- 建立搜索索引并计算匹配节点。
- 类型筛选。
- 全局、社区聚焦与搜索状态下的节点和边集合。
- 验证既有选中节点是否仍在当前可见集合中。

结构索引和文本搜索索引是两种不同契约：结构索引按稳定 ID 查找图谱对象，文本搜索索引只服务文字匹配，不能混成同一个无类型字典。

现有 `model/visibility.ts` 是文本索引、匹配规则和语义可见集合的唯一负责人，但“唯一负责人”不等于本轮把两种现有搜索范围合并为一种：

- 常规绘制搜索继续使用原始 JavaScript 真假值选择“标签或 ID”，并匹配正文前 500 个 UTF-16 代码单元。空白字符串标题仍会阻止 ID 回退，数值 `0` 标题仍会回退到 ID；不能在本轮顺手改成修剪后判断、Unicode 字素截取或“标签与 ID 总是都可搜索”。
- Atlas 可见快照搜索继续匹配标签、ID、类型文案、来源路径、摘要和清理后的完整正文。

两种规则必须使用能说明范围的不同名称、分别固定迁移前后输出，不能以去重为由改变任一结果。常规搜索消费归一化阶段一次生成的兼容投影，不回头读取原始 `GraphData`。现有 `render/search.ts` 只消费常规绘制搜索结果，负责界面所需的搜索状态、命中/淡化标记与上一个/下一个结果导航；不得复制文字归一化或匹配规则。

该模块只决定“哪些图谱对象在语义上可见”，不决定世界位置、屏幕坐标或具体画法，也不接管 `select/` 与 facade 已经拥有的选择语义。它只检查既有选择在筛选后是否仍有效。

### 6.4 初始布局

负责：

- 从归一化图谱模型产生确定性的初始世界位置。
- 社区中心和节点相对位置。
- 只由世界位置决定的 `layoutBounds` 与布局结果。

布局结果必须以带类型的位置映射返回，不得原地改写归一化模型中的节点。它不处理 Pin 与实时拖动覆盖、`contentBounds`、`framingBounds`、相机缩放、屏幕尺寸、DOM 事件或宿主回调。钉扎文件的现有归一化继续由已经独立的布局文件模块负责，不重新并回本模块。

### 6.5 视角控制

负责：

- 视角归一化和边界限制。
- 缩放、平移、适配窗口与窗口变化后的锚点保持。
- 世界坐标、屏幕坐标和小地图之间的转换。

现有 `render/viewport.ts` 已经承担大部分相机和投影职责。本次只把它仍向旧工具箱借用的有效规则收回并类型化；只服务早期模板、当前路线不再使用的旧视角函数不重新迁移。它消费已经计算好的 `framingBounds`，不负责决定社区可见集合、标签预算或密度策略。

### 6.6 视口感知的绘制策略与快照组装

绘制策略负责把归一化模型、语义可见集合、初始布局、位置覆盖和视口信息组合成现有绘制规则，包括：

- 按既有优先级得到最终位置与 `contentBounds`，再由 `contentBounds` 和视口比例得到 `framingBounds`。
- 社区质量、聚焦规模和呈现等级。
- 类型筛选、临时对象显示与最终绘制集合合并。
- 密度与节点显示模式、标签预算、边预算和交互预算。
- 节点与边优先级、稳定骨架选择、社区节点层级、社区边层级和标签方向。
- 当前社区阅读中与屏幕宽度相关的标签数量。
- Sigma 与 DOM/SVG 共用的可绘制规则快照。

边路径、线宽、透明度等单对象纯计算回到现有边或几何负责人；社区 wash、局部地图几何等专属纯计算回到现有社区负责人。它们不需要第三个新增文件。`render/render-policy.ts` 只收纳需要同时观察模型、可见集合、位置或预算的跨对象纯规则，不能成为无边界的杂项文件。

当前实现存在旧 `resolveAtlasVisibleSnapshot` 预算和 `render/model.ts` 绘制预算两级处理。迁移时先保持两级执行顺序和最终结果，不能顺手合并；只有另一个经过确认的行为优化任务才能改变预算算法。

`render/model.ts` 最终只负责调用各负责人并组装结果，不再归一化原始数据、不私藏另一套图谱模型、不直接实现布局或搜索规则。选择与聚合行为继续由现有负责人从兼容 `GraphData` 解析，不并入共享绘制模型。适配层接收“可绘制快照”和“已经解析的选择/聚合行为”两路结果并组合；不得为绘制目的重新读取兼容节点或边。Sigma 与 DOM/SVG 只能经适配层消费组合后的结果，不能直连模型内部。

本次不新增跨更新长期缓存，也不设计新的版本或失效协议。每次更新内，归一化、初始布局、实际需要的两种搜索索引和绘制策略各最多计算一次并向后共享；现有界面搜索索引复用行为保持不变。

### 6.7 存储、收藏、笔记队列与旧学习出口

这些能力先按真实使用面判断，不机械搬迁：

- 当前宿主仍使用的运行时行为，迁到清楚的负责人并补类型。
- 只被早期 wash 模板和专属测试使用的存储、收藏、笔记队列出口，随模板退休删除。
- 旧 `getWikiStorageNamespace` 虽无直接运行时导入，但现行离线生成脚本复制了同一算法来定位 Pin 与主题记录；该兼容契约归离线宿主所有，删除旧出口前必须把固定输入与预期存储名称测试迁到现行离线生成或浏览器回归。
- 旧 learning 中仍影响当前图谱输入兼容的规则，收进归一化图谱模型内部。
- 没有真实消费者的公共导出和测试一起删除。

### 6.8 职责落点与依赖方向

实施时优先把真实实现放回现有负责人，不按概念数量机械创建文件。本次只新增两个实现文件：`src/layout/initial-layout.ts` 承接初始世界布局，`src/render/render-policy.ts` 承接取景、密度与预算规则；其余职责充实现有入口。最终名称可以按现有目录惯例小幅调整，但不能增加第三个实现文件，也不能为了保持一张整齐的模块图继续拆分。

| 职责落点 | 处理范围 | 允许依赖 | 禁止依赖 |
|---|---|---|---|

## 7. 对外入口与类型契约

迁移期间继续使用现有分类入口，不再增加一层新的总入口。

这里的“继续使用”只保证仓库其他包、离线脚本和受支持入口的兼容。图谱引擎内部迁移完成后改为直接引用各职责负责人，不把 `model/index.ts` 当作内部服务定位器。

现有对外便利入口若仍接收 `GraphData`，可以保留调用形状，但内部必须先分别得到可绘制快照与选择/聚合结果，再交给适配映射；不能用兼容入口掩盖适配层重新推导绘制事实。

兼容保证适用于当前受支持的引擎入口，以及工作台和离线宿主正在使用的行为。开始迁移前，从工作台、服务端、离线启动脚本和现有公共入口测试生成一份明确出口清单，并由边界测试固定。

至少保留以下契约：

- `createGraphEngine` 与当前 `GraphEngine` 生命周期、选项和能力回调。
- `createGraphWorkbenchCapabilities`、`createGraphOfflineCapabilities` 和 `createGraphStandaloneCapabilities`。
- 工作台和服务端正在导入的 GraphData、布局、Pin、选择、摘要、可见性和阅读动作类型及函数。
- 离线启动脚本使用的 `normalizeGraphLayoutFile` 与 `normalizeGraphPinMap`。
- 服务端使用的 `diffGraphData`。
- 现有测试明确保护的 `createGraphRenderer`、`createStaticGraphRenderer` 及其等价关系。

图谱引擎包是仓库内私有包；只被废弃模板或旧测试引用的 `defaultQueue`、旧存储、旧 learning、旧 Atlas 视角等历史导出不构成长期外部兼容承诺，可以在完成使用面审计后删除。不能仅凭“包是私有的”删除仍被工作台、服务端、离线启动脚本或当前公共入口测试使用的符号。

迁移期间允许新旧实现短暂并存，但有两个约束：

1. 旧文件的直接引用清单只能减少，不能增加。
2. 最终状态不保留旧名称转发壳，除非该名称被证明属于当前受支持入口。

新的共享类型应直接描述归一化模型、布局结果和可见快照。下游不得通过 `Record<string, unknown>`、无边界的 `any` 或重复本地类型掩盖真实契约。

严格类型只约束内部实现，不替代运行时防御。原始 JSON、旧知识库数据和宿主传入值先经过运行时安全投影，再进入 facade、选择、摘要和绘制归一化；不能因为 TypeScript 声明完整就直接信任外部输入。

## 8. 迁移策略

采用“先并存、逐块迁移、持续验证、最后删除”的方式。每一步都保持仓库可构建、可测试、可运行。

### 8.1 固定基线与边界

- 固定文字、搜索、模型、布局、可见快照和视角计算的现有输出。
- 在迁移任何实现前先给两套 Sigma 试验接入专属 `hover_preview` 动作、范围校验和自动比较器；在未改动旧实现的同一提交上连续运行三次，保存去除本机路径后的原始结果与中位数清单，作为不可由新实现重写的性能基线。
- 记录当前直接引用旧工具箱的位置，并建立只能缩小的允许清单。
- 通过当前公开入口验证行为，避免把旧文件路径本身固化成新契约。
- 保留现有 Sigma、DOM/SVG、工作台和离线图谱基线。

### 8.2 迁移文字与搜索

- 先迁移依赖少、已有测试密集的文字规则。
- 再迁移搜索索引与匹配规则，并移除 `render/search.ts` 对旧文件的直接引用。
- 每个迁移后的分类模块拥有真实实现和具体类型，不再只是转发。

### 8.3 迁移归一化图谱模型

- 为节点、边、社区、起始节点、洞察和索引定义唯一内部类型。
- 逐项迁移旧字段兼容、默认值、排序和过滤规则。
- 让绘制模型直接消费带类型结果，删除重复的 Atlas 临时类型和强制转换。

### 8.4 分离布局、可见性与视角

- 初始世界位置与 `layoutBounds` 进入布局负责人。
- 布局返回独立位置映射，不再通过修改归一化节点传递结果。
- 搜索、筛选和聚焦进入语义可见范围负责人。
- Pin 与实时拖动位置按现有优先级形成最终位置和 `contentBounds`；`framingBounds`、密度和两级预算进入视口感知的绘制策略负责人。
- 社区质量、聚焦等级、类型筛选、临时对象合并、优先级、骨架与局部地图层级等纯策略一并离开 `render/model.ts`；边和社区专属纯计算优先回到现有负责人。
- 相机边界、缩放、投影和小地图进入视角负责人。
- 更新架构契约和依赖边界测试，保证组装层与两条渲染路线不重新吸收这些规则。
- 迁移过程中保持现有坐标、实时位置与 Pin 优先级、范围留白、顺序、两级预算顺序、视口宽高比锁定和最终边界结果不变。

### 8.5 清理无消费者能力与早期模板

- 全仓核对存储、收藏、笔记队列和旧 learning 出口的直接消费者与复制实现，不能只靠 import 搜索判断死亡代码。
- 保留仍影响当前输入兼容的内部规则，删除无消费者出口。
- 将离线 Pin 与主题存储名称的稳定性测试迁到现行离线宿主后，再删除旧存储出口及其测试。
- 模板退休作为独立收尾阶段和独立提交，不与共享模型迁移混在同一个提交中。
- 按 Skill 维护区规则，迁出仍有效的主题或契约后，删除早期 wash 模板、专属测试和构建脚本中的复制、加载逻辑；保留或改名为通用的过期资源清理，用于删除已有知识库中的旧 wash 文件。
- 删除清单按真实路径与消费者确认，不能按文件名包含 `wash` 批量删除现行社区 wash 代码或浏览器回归。
- 模板退休提交必须通过安装 dry-run、当前离线图谱生成和 Skill 专用隐私检查。

### 8.6 删除旧工具箱

- 将旧工具箱直接引用清单降为零。
- 删除旧文件和所有过渡转发。
- 核对受影响代码旁的路线说明与 ASCII 数据流图，修正“Sigma 不经过共享绘制模型”等已经过时的描述；不保留会误导后续维护者的旧阶段注释。
- 加入长期边界检查，禁止重新出现旧路径、`@ts-nocheck` 和无类型公共结果。
- 完成全量自动检查与真实浏览器验收。

## 9. 算法与问题发现规则

迁移时不主动改变算法，但发现的问题不能被忽略。实施前建立一张独立的“#159 实施期间图谱后续问题”远端追踪 issue，作为后续问题的统一入口；不把 #159 本身变成长期问题收集箱。该追踪 issue 只是登记入口，不作为 #159 的子任务或阻塞项；若本轮没有发现可延期问题，保持空清单即可，不影响 #159 完成。

发现按两类处理：

### 9.1 必须在 #159 内修复

满足任一条件时，不得推迟：

- 问题由本次迁移造成。
- 迁移前后结果不一致。
- Sigma、DOM/SVG、工作台或离线图谱出现语义漂移。
- 会导致本规格验收失败。
- 会让旧工具箱无法安全删除。

这类问题必须在发现它的迁移阶段修复并补防退化测试。

### 9.2 可以进入后续追踪

只有同时满足以下条件的问题才能后续处理：

- 问题在本次迁移前已经存在。
- 它不是本次结构清理的完成条件。
- 保持现有行为不会损害数据安全或让当前图谱无法使用。
- 当前工作可以在不掩盖问题的前提下继续。

登记方式保持简单：

1. 先搜索远端是否已有相同问题。
2. 新问题单独创建 issue，写清发现、影响、证据和不属于 #159 的原因，并先标记 `needs-triage`。
3. 新 issue 作为追踪 issue 的正式子 issue。
4. 若已有 issue 已属于其他父级，只建立双向链接，不改写原父子关系。
5. 追踪 issue 和后续问题默认不阻塞 #159，也不要求这些后续问题在本轮解决。

不引入额外项目、里程碑或新的标签体系。

设计审查已确认“重复 ID 与自动生成 ID 碰撞的长期治理”是首个符合后续处理条件的既有问题。建立专门追踪 issue 后，为它搜索或创建独立远端 issue，并关联迁移前碰撞基准；#159 只负责保持现有结果。

## 10. 错误处理与兼容

迁移必须保留现有宽容边界：

- 缺失节点、边端点、社区或可选字段时，沿用现有安全默认值和过滤规则。
- 未知节点类型、关系类型和置信度继续归入现有兼容值。
- 空图谱、无社区图谱和旧 learning 数据继续得到确定结果。
- 文字分段在 `Intl.Segmenter` 不可用时继续保护组合字符和 emoji。
- 存储不可用时，当前仍受支持的离线行为不得崩溃。
- 不完整或畸形数据必须先由总函数生成结构安全的兼容 `GraphData` 与搜索兼容信息，再稳定生成共享可绘制快照；不能依赖切换渲染器来修复共享数据错误。
- DOM/SVG 回退边界从共享快照成功之后开始，只接住 Sigma 专属的 Graphology 转换、WebGL、canvas 初始化或运行故障。
- 共享模型、布局、绘制策略或快照组装若因程序错误失败，DOM/SVG 会重复同一故障，不得伪装成可回退场景；测试必须直接失败，运行时进入现有错误状态。
- 工作台在首次创建引擎和后续 `setData` 更新两处捕获共享层失败，销毁或清空失败实例后复用现有 `applyGraphFailure`，使数据、选择、可见状态与 Pin 按现行规则归零并显示现有错误状态；不能留下仍可交互的半成品图谱。
- 离线单文件入口捕获引擎首次创建失败并复用现有 `showError` 与 `.offline-error` 区域；不能新增另一套错误容器，也不能让异常中断后只留下空白画布。
- 数据不完整导致离线 HTML 无法启动时，继续显示现有错误提示。

本次不新增用户可见错误类型，也不改变错误文案和恢复路径。

## 11. 测试与验证

### 11.1 分步自动测试

每迁移一个职责，立即运行其专门测试，并增加以下保护：

- 在迁移第一个函数前建立 `packages/graph-engine/test/fixtures/issue-159/` 行为基准和独立的迁移一致性测试。基准由旧实现通过当前受支持入口一次性生成、人工核对并提交；新实现不得自动刷新预期值。
- 同一批输入必须逐字段精确比较迁移前后输出；Map、Set 等结构先转成稳定数组，不能只比较数量、抽样字段或“没有抛错”。
- 明确输入输出类型的契约测试。
- 旧字段、缺字段、未知值、空集合和异常输入测试。
- 直接引用旧工具箱的边界测试，允许清单逐步缩小到零。
- Sigma 与 DOM/SVG 消费同一共享模型结果的一致性测试。

现有 `architecture.test.ts` 继续验证负责人说明，但不能单独充当依赖门禁。新增或扩展边界测试时，通过 TypeScript 模块解析得到生产源码的 import 与 re-export 关系，至少验证：

- 旧工具箱路径的 import、动态 import 与 re-export 只能减少并最终归零。
- 图谱引擎内部不得经 `model/index.ts` 汇总入口隐藏真实依赖。
- 模型、布局、绘制策略、视角、适配与两条渲染路线遵守目标依赖方向，没有禁止的反向引用。
- Sigma 与 DOM/SVG 路线不能绕过适配与可绘制快照直连模型内部。

`@ts-nocheck` 等禁止标记可以做文字扫描；真实模块依赖不能只靠正则搜索 import 文本。

本次至少补齐或改写以下测试：

- `model-migration-parity.test.ts` 与 `test/fixtures/issue-159/`：逐项执行并核对迁移前行为基准。
- `tsconfig.type-tests.json` 与 `test-types/`：单独编译新的归一化模型、布局结果、可见快照和绘制策略公开类型。正例必须通过，反例使用 `@ts-expect-error` 证明错误字段、错误阶段输入和越界调用会被拒绝；构建后再编译一个只消费 `dist/index.d.ts` 的最小样例，确认发布声明与源码契约一致。相关命令纳入包级 `typecheck`、`build` 后验证和全仓质量检查。
- 运行时投影、模型与可见性测试：覆盖 `unknown`、非数组节点/边、残缺对象、无效端点、重复节点/边/learning 社区 ID、缺失 ID 与同名真实 ID 撞车，以及只在常规搜索或只在 Atlas 搜索命中的字段；固定兼容 `GraphData`、数组顺序、最后写入索引、度数、社区成员、起点和端点过滤结果，并证明两种搜索范围没有被合并。

- 搜索边界测试：固定“普通标签不额外匹配 ID、空标签回退 ID、纯空格标签不回退 ID、数值 `0` 标签回退 ID、正文第 500 个 UTF-16 代码单元可匹配而第 501 个不可匹配、代理对跨 500 边界时保持旧切片结果、Atlas 搜索仍可匹配完整清理正文”的现有差异。
- 初始布局测试：断言返回独立位置映射，调用前后的归一化节点深度相等；已有明确坐标、缺坐标和超出旧百分比范围的相对形状保持不变。
- 绘制策略测试：同一节点同时存在实时位置与 Pin 时验证“实时位置 > Pin > 初始布局”，并逐级移除覆盖验证回退顺序；`contentBounds` 包住远端位置，社区 wash 上限不随远端节点失控扩大。
- 范围语义测试：固定当前全局与社区聚焦状态下用于计算范围的节点集合，包括“远端节点位于另一个社区”的案例，防止迁移时无意改变取景对象。

## 13. 风险与控制

### 类型收紧暴露旧数据歧义

控制方式：在归一化模型边界集中处理运行时输入，不把 TypeScript 类型当作外部数据可信证明；为旧字段和未知值补明确测试。

### 拆分过程中改变顺序、默认值或预算

控制方式：先固定现有输出，再逐块迁移；节点、边、社区、布局、密度和搜索结果都做前后对比。

### 把仍有消费者的代码误判为无用

控制方式：同时检查生产源码、构建脚本、生成产物、浏览器入口和仓库测试；当前路线的真实浏览器验收通过后才删除。

### 旧模板退休误删仍有效资产

控制方式：先确认主题、样式和数据契约的当前负责人；仍有效部分先迁移，之后删除整个旧模板目录和专属测试。

### 一次改动范围过大

控制方式：保持现有入口，分职责迁移，每一步都可构建、可测试、可运行；只有全部消费者迁完后才删除旧文件。

### 后续问题追踪变成无边界杂物箱

## 14. 失败场景与保护

| 真实失败场景 | 测试保护 | 运行时处理 | 用户结果 |
|---|---|---|---|
| 旧字段或畸形节点、边让 facade、选择、摘要或绘制抛错 | 运行时投影、入口流程与共享快照契约测试 | 宽容投影；真正的共享层程序错误进入宿主现有错误状态 | 得到结构安全的兼容数据并继续，或看到明确错误；不做无效 DOM 回退 |
| 迁移改变默认值、顺序、坐标或预算 | 不可自动重写的迁移前行为基准逐字段比较 | 不发布不一致产物 | 变化在提交前被拦住，用户无感 |
| 初始布局原地修改归一化节点 | 深度不可变测试与独立位置映射类型 | 布局只返回位置映射 | 后续搜索、选择和摘要不会读到被布局污染的事实 |
| 实时位置或 Pin 位于初始范围外 | 优先级、`contentBounds`、远端其他社区节点和浏览器几何测试 | 最终范围包住位置覆盖 | 节点仍在可取景范围内，社区 wash 不失控扩大 |
| 两种搜索范围被误合并 | 第 500/501 字符、标签/ID 回退和 Atlas 完整正文对照测试 | 保留两个具名匹配入口 | 搜索结果与迁移前一致 |
| 适配层重新从原始数据推导绘制事实 | 双输入契约、真实 Sigma/DOM 消费一致性和依赖边界测试 | 适配只组合既有结果 | 两条路线的对象、位置、选择与预算一致 |
| Sigma 专属转换、WebGL 或 canvas 失败 | 在共享快照之后注入故障，并运行浏览器强制 Sigma 失败流程 | 切换 DOM/SVG 回退 | 自动进入可操作的故障回退 |
| 共享快照在工作台首次创建或后续更新、离线首次创建时抛出程序错误 | 宿主入口捕获、失败实例清理和工作台/离线浏览器错误接线测试 | 工作台复用 `applyGraphFailure`；离线复用 `showError` | 显示现有错误状态；不错误回退或留下半成品图谱 |
| 离线存储名称变化 | 固定 wiki 标题与路径的精确存储键回归、刷新读取测试 | 继续使用旧名称算法 | 既有 Pin 与主题记录继续可读 |
| 删除 wash 模板后主回归仍调用旧测试，或已有知识库残留旧文件 | 完整运行 `tests/regression.sh`；预置旧文件后重新生成 | 通用过期资源清理删除遗留文件 | 安装包不加载旧资源，升级后的知识库目录也不残留废弃文件 |
| 拆模块导致同一更新重复建模或索引，或大图悬停明显变慢 | 调用契约测试、必需的 `hover_preview` 动作与 1k/5k/10k 大图性能基线 | 超过门禁就停止合入并定位重复计算 | 现有性能阈值、内存和交互预算继续通过，悬停退化在提交前被拦住 |

上述场景在实施前都有明确测试或错误处理要求；按本规格完成后不保留“无测试、无处理且静默失败”的关键路径。

## 15. 实施依赖与并行顺序

| 阶段 | 主要范围 | 依赖 |
|---|---|---|
| A. 固定基线与出口 | 行为 fixture、悬停测量与三次性能基线、出口清单、依赖门禁、类型测试入口 | 无 |
| B. 文字与两种搜索 | `model/labels`、`model/visibility`、`render/search` | A |
| C. 模型归一化 | `model/atlas`、learning 兼容、结构索引 | A；与 B 约定公共文字类型 |

| F. 真实宿主验收 | Sigma、DOM/SVG、工作台、离线 HTML、大图性能 | E |
| G. 历史出口与模板退休 | 旧出口、wash 模板、专属测试、构建和回归清理 | F |

并行安排：A 先单独完成；随后 B 与 C 可并行，但 `model/index.ts` 和共享类型由 C 统一收口，避免重复修改。C 完成后 D 开始；E 依赖 B、C、D，顺序执行。E 完成后，工作台浏览器验收、离线浏览器验收和性能测量可分三路并行。G 必须最后执行，并在合并所有验收结果后删除旧文件。

冲突提醒：B 与 C 都触及 `model/`，不得各自改同一汇总出口；D 与 E 都可能读取位置类型，但只有 D 定义布局结果，E 只消费。模板退休不能与仍在迁移旧能力的分支并行删除源文件。

需要维护的源码内图示保持克制：`src/architecture.ts` 记录模块依赖方向，`src/render/model.ts` 在组装入口保留“模型 -> 布局/可见性 -> 策略 -> 快照”的短图，`src/facade.ts` 在路线切换处记录“共享成功后才允许 Sigma -> DOM/SVG 回退”的边界。纯文字、几何和搜索函数不加重复图示。改动这些流程时，图示与代码在同一提交更新。
