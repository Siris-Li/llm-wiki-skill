# 统一社区抽屉设计

日期：2026-06-28  
范围：llm-wiki-agent 图谱右侧抽屉、全局 Sigma 选择入口  
相关 issue：#71、#74

## 背景

全局图谱从 DOM/SVG 全图迁移到 Sigma 全局图后，社区点击形成了两套右侧抽屉心智：

- 点击普通分组社区：打开社区简介抽屉，包含节点数、核心节点、搜索命中、固定节点、桥接关系和“进入社区”。
- 点击“未分组”：因为 `_none` 虚拟社区没有被摘要层识别，漏到底层选区抽屉，显示多页选区、探索潜在关系、对话输入、发送和新对话。

这不是单纯的 bug。未分组漏出的选区抽屉保留了早期 DOM 架构下有价值的能力：对一组选中的页面提问、探索潜在关系、进入当前对话或新对话。问题在于普通社区和未分组社区右侧抽屉形态不一致，用户点击不同社区时像切换了两个产品。

本设计将两套抽屉融合为一个统一社区抽屉：社区概览负责看清结构，固定动作区负责选择工作流，核心节点区负责快速理解内容，对话区负责把选区带入 agent。

## 设计目标

1. 点击任何社区后，右侧抽屉结构保持一致，避免普通社区和未分组社区心智割裂。
2. 保留现有社区简介的“看这个社区是什么”能力，但去掉冗余的大块信息。
3. 保留早期选区抽屉的对话能力，并让它成为所有社区共享能力。
4. “进入社区”作为视图导航，放到抽屉顶部，用户点社区后能立刻看到。
5. “探索潜在关系”对所有社区一直可见，但普通社区为次级动作，未分组为推荐动作。
6. 恢复全局 Sigma 路径下 Shift 多选和 +邻居入口，让 ADR-21 的四式选择重新闭环。

## 非目标

1. 不重新设计社区阅读视图。
2. 不改变图谱社区、颜色、连线、节点归属的数据来源。
3. 不引入自由框选或套索选择。
4. 不改变 `/api/prompt` 文本通道，不新增专用图谱问答接口。
5. 不把桥接关系作为首屏重点信息；它仍可保留为次级查看能力。

## 产品原则

### 抽屉身份

右侧抽屉的统一身份是：

> 看清这组内容，并马上对它提问。

它不是纯简介页，也不是纯操作面板。点击社区后，用户常见意图有三类：

- 想看这个社区是什么。
- 想直接进入社区阅读。
- 想让 agent 基于这组页面做事。

统一社区抽屉必须同时支持这三件事，但要分层清楚。

### 动作分层

“进入社区”是导航动作，不是 agent 动作。它放在顶部标题区右侧。

“总结这一簇 / 找知识缺口 / 生成主题页 / 探索潜在关系”是 agent 动作，放在中部固定动作区。

“补充说明 / 发送 / 新对话”是对话动作，放在底部对话区。

## 抽屉结构

### 1. 顶部概览区

顶部概览区替代当前过重的社区简介首屏。

内容：

- 类型标识：社区。
- 社区名称。
- 顶部右侧“进入社区”按钮。
- 一句简短说明。
- 关键统计：页数、核心数、链接数、孤立数。
- 轻量标签：固定状态、搜索命中状态、结构状态。

普通社区说明：

> 这组页面围绕同一主题聚在一起。你可以先看结构，也可以直接让 agent 基于这一组页面继续工作。

未分组说明：

> 这些页面暂未形成明确社区。你可以让 agent 探索它们之间是否存在潜在关系。

未分组顶部不显示“进入社区”，只显示“暂未形成明确社区”的说明。原因是未分组不是一个真正的社区阅读空间，强行进入会继续制造视图心智割裂。

### 2. 固定动作区

动作区固定显示四个动作，位置不随社区变化大幅跳动：

1. 总结这一簇
2. 找知识缺口
3. 生成主题页
4. 探索潜在关系

推荐态根据社区类型变化：

- 普通社区：推荐“总结这一簇”。
- 未分组：推荐“探索潜在关系”。
- 弱结构社区：可推荐“找知识缺口”或“探索潜在关系”，但按钮位置仍保持固定。

“探索潜在关系”对普通社区也一直可见，作为次级动作。这样用户知道任何一组页面都可以继续找潜在连接，不会因为切换社区而丢失这个能力。

### 3. 核心节点区

核心节点区只展示少量节点，默认最多 3 个。超过 3 个时显示“查看全部”。

每个节点行只保留：

- 节点标题。
- 简短角色标签，例如核心、主题、相关。

点击节点仍切换到节点摘要，不自动进入社区阅读。

### 4. 对话区

对话区融合旧选区面板能力。

内容：

- 补充说明输入框。
- 发送。
- 新对话。
- 轻提示：选区会带入对话。

发送时沿用现有选区展开机制，把选区摘要、页面清单、内部链接、社区归属、用户补充说明展开进 prompt。对话历史继续保持胶囊态，不把长结构文本直接刷屏。

## 需要优化掉的信息

当前社区简介中以下内容不再作为首屏大块展示：

1. 搜索命中空状态。
   - 没有搜索时不显示独立大块。
   - 有搜索时用轻标签提示，必要时在核心节点区附近显示命中列表。

2. 固定节点空状态。
   - 无固定时只显示轻标签。
   - 有固定时可显示为核心节点区的一个过滤视图或次级列表。

3. 长桥接关系列表。
   - 默认不铺开。
   - 保留为“查看桥接关系”次级入口，或在动作结果中由 agent 解释。

4. 重复社区标签。
   - 社区名已经是标题，不再重复显示社区 id chip。

5. 底部“进入社区”按钮。
   - 移到顶部标题区右侧。

## 选择入口设计

### 普通社区点击

用户点击普通社区云团、标签或图例：

1. 停留在 Sigma 全局图。
2. 当前社区高亮，其他社区弱化。
3. 右侧打开统一社区抽屉。
4. 顶部显示“进入社区”。
5. 固定动作区推荐“总结这一簇”。

### 未分组点击

用户点击“未分组”：

1. 停留在 Sigma 全局图。
2. 未分组节点高亮。
3. 右侧打开统一社区抽屉。
4. 顶部不显示“进入社区”，只显示“暂未形成明确社区”的说明。
5. 固定动作区推荐“探索潜在关系”。
6. 统计重点显示孤立数和链接数，帮助用户理解为什么建议探索。

### 单节点点击

单节点抽屉保持节点摘要，但需要明显提供“+邻居”入口。

“+邻居”触发后打开统一选区抽屉，选区包含该节点及一跳邻居。动作区根据选区事实推荐动作。

### Shift 多选

全局 Sigma 图恢复 Shift 多选：

1. Shift+点击节点时，累加或移除节点。
2. 多选结果打开统一选区抽屉。
3. 抽屉结构与社区抽屉一致，但顶部标题显示选区名称而不是社区名称。
4. 若多选节点跨多个社区，动作仍按现有结构事实生成。

## 状态与数据规则

### `_none` 统一映射

渲染层和选择层已经把未分组节点映射为 `_none`。摘要层也必须认识同一个虚拟社区。

统一规则：

- `_none` 表示未分组虚拟社区。
- 节点筛选时，`node.community == null` 或空字符串归入 `_none`。
- `_none` 的显示名固定为“未分组”。
- `_none` 没有真实 community 记录时，也能生成统一社区抽屉 payload。

### 结构状态

抽屉可根据结构事实显示轻量结构状态：

- 结构清晰：链接数大于 0，孤立数较少。
- 结构松散：链接数较少，孤立数较多。
- 暂未成组：`communityId === "_none"`。

结构状态只影响提示和推荐态，不改变按钮布局。

## 组件边界

### 图谱引擎

负责：

- 产生 selection。
- 支持 `node`、`community`、`neighbors`、`nodes` 四类选择。
- 在 Sigma 全局路径传递 Shift 选择意图。
- 保持社区高亮和选择状态同步。

不负责：

- 决定右侧抽屉长什么样。
- 拼 prompt 文本。
- 理解工作台会话。

### 工作台前端

负责：

- 把 selection 转成抽屉状态。
- 渲染统一社区抽屉和统一选区抽屉。
- 执行动作按钮和对话发送。
- 处理“进入社区”导航命令。

### 摘要层

负责：

- 为真实社区和 `_none` 未分组社区生成统一 payload。
- 提供核心节点、统计、结构状态、推荐动作所需数据。
- 不再把 `_none` 作为 missing community 兜底。

## 错误与空状态

1. 社区没有节点：
   - 显示不可用状态，不进入对话抽屉兜底。

2. 未分组没有节点：
   - 不显示未分组社区入口。

3. 没有核心节点：
   - 核心节点区显示“暂无核心节点”，但不影响动作区。

4. 没有搜索命中：
   - 不显示独立大块，只在概览标签中提示或完全隐藏。

5. 没有固定节点：
   - 不显示独立大块，只在概览标签中提示。

## 测试要求

### 单元测试

1. `_none` 未分组社区可以生成社区抽屉 payload。
2. 普通社区和未分组社区都走统一社区抽屉状态。
3. 普通社区动作区包含“探索潜在关系”，但推荐态不是它。
4. 未分组动作区推荐“探索潜在关系”。
5. 搜索命中和固定节点空状态不生成独立大块。
6. 单节点“+邻居”入口触发 `neighbors` selection。
7. Sigma 全局 Shift+点击产生 `nodes` selection。

### 浏览器验证

1. 点击普通社区：抽屉顶部出现“进入社区”，中部固定动作区可见，底部对话区可见。
2. 点击“进入社区”：进入 DOM/SVG 社区阅读视图。
3. 点击未分组：抽屉结构一致，但推荐“探索潜在关系”，不出现普通“进入社区”导航。
4. 点击普通社区后再点未分组：抽屉结构不大变，只更新内容和推荐态。
5. Shift 多选两个节点：打开统一选区抽屉，并可发送到当前对话。
6. 单节点点击“+邻居”：扩展为一跳邻居选区。
7. 搜索命中为空和固定节点为空时，首屏不出现大块空卡。

## 验收标准

1. 普通社区和未分组社区不再出现两套右侧抽屉心智。
2. 用户点击社区后，不滚动即可看到“进入社区”或未分组说明。
3. 用户点击任何社区后，都能在同一抽屉里看到固定动作区和对话区。
4. “探索潜在关系”对所有社区可见，对未分组为推荐动作。
5. Shift 多选和 +邻居在全局 Sigma 路径可用。
6. 现有节点摘要、社区高亮、进入社区阅读不回归。

## 实施顺序建议

1. 先让摘要层支持 `_none` 未分组社区。
2. 抽出统一社区抽屉 view model，明确概览、动作、核心节点、对话区数据。
3. 改造社区抽屉 UI，移动“进入社区”到顶部。
4. 接回 Sigma 全局 Shift 多选。
5. 补单节点 +邻居入口和统一选区抽屉入口。
6. 跑单元测试和浏览器验证。
