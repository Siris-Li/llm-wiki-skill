# Sigma 社区阅读模式迁移设计

> 状态：设计稿，待用户复核
> 日期：2026-07-04
> 关联：Issue #95、Issue #93、PR #94、PR #96、ADR-21/22/23/26
> 取代：`2026-07-03-community-enter-transition-design.md`（旧过渡动画稿已备份到桌面并从仓库删除）

## 0. 给体验复核看的结论

如果只从用户体验判断，这份设计要确认 8 件事：

1. 全局图仍负责“找社区”：点社区后先看摘要，再由“进入社区”深入。
2. 点“进入社区”后，画布马上成为主角：节点居中，不再保留社区简介抽屉。
3. 社区阅读只看当前社区内部节点和关系，不把周边社区作为背景塞进来。
4. 节点位置要延续全局里的相对关系，不突然变成另一张图。
5. 社区颜色全程保持身份一致：全局、选中高亮、进入社区后都不把节点改成另一种状态色。
6. 社区很密时，系统先减少标签、弱化次要边、稍微拉开视野，不默认自动重排节点；用户手动拖动节点整理空间的能力要保留。
7. 用户进入社区后可以直接 hover 看关系、点节点读内容；社区多选作为紧随首版的必做切片。
8. 返回全局后，仍能看出刚才是从哪个社区回来的。

你不需要先懂 DOM、Sigma 或图谱引擎。只需要看最终体验是否符合一句话：**进入社区后，是在同一张图里深入阅读，而不是跳到另一个图谱页面。**

## 1. 一句话目标

把“进入社区”后的主路径从 DOM 社区视图迁到 Sigma，让图谱只有一套主画布逻辑：

- 全局 Sigma：发现社区、选择社区、看整体结构。
- Sigma 社区阅读模式：只看当前社区内部节点和关系，直接 hover / 点击节点阅读。

这次不再优先做“Sigma 到 DOM 的过渡动画”。动画只能让两套画布切换更顺，但不能解决每次图谱功能都要维护两套方案的问题。

## 2. 当前判断

### 2.1 旧设计为什么要废弃

旧 Phase 2.1 的中心是“全局 Sigma 和 DOM 社区之间切得更顺”。现在问题已经变了：

- 用户真正不舒服的不是切换动画慢，而是进入社区后像换了一套图谱。
- DOM 社区和 Sigma 全局天然会有位置、缩放、标签、hover、边线、抽屉联动差异。
- 继续补过渡动画，只是让分裂感变得温柔一点，不会减少长期维护成本。

所以新方向应从“修补两套画布”改为“社区阅读也回到 Sigma 主系统”。

### 2.2 现在迁移时机比 #95 创建时更成熟

#95 当时被放到远期，主要是为了不吞掉 Phase 2，不是因为 Sigma 方向错误。

PR #96 已经把社区近景最重要的规则抽到共享层：

- 节点基础位置。
- 核心 / 相关 / 外围分层。
- 标签预算。
- 边线骨架 / 相关 / 背景分层。
- 来源社区上下文。
- 社区阅读态冻结自动漂移。

这意味着迁移不是从零重写 DOM 社区，而是让 Sigma 消费同一批社区阅读规则。

## 3. 产品体验

### 3.1 全局仍然是“找社区”

全局 Sigma 保持现在的心智：

1. 用户点击社区。
2. 全局进入社区高亮态。
3. 右抽屉展示社区摘要和动作。
4. 用户明确点击“进入社区”。

这里不改“先看摘要，再进入”的全局决策点。它仍然是用户决定是否深入这组页面的入口。

全局社区高亮态仍然是地图导航层，不是单社区隔离层。用户可以直接点击其他社区标签或社区区域切换当前高亮社区，不需要先“回全图”再选择。标签和区域是同一个切换动作：仍停留在全局高亮态，只切换高亮对象，并把右侧社区摘要同步到新社区，不自动进入社区阅读模式。

因此，未选中社区的社区级标签要保留：它们是地图上的路标，用于快速切换社区。退后的是未选中社区的节点细节、节点标签和内部关系网，不是社区标签本身。

未分组不是一个真实社区，不提供“进入社区”。它继续留在全局摘要和选区动作里处理，避免把一堆未成组节点强行做成阅读空间。

### 3.2 进入社区后直接开始读关系

点击“进入社区”后：

- 画布进入 Sigma 社区阅读模式。
- 默认只显示当前社区的节点和社区内部关系；内部关系保留结构感，但背景边要明显退后。
- 不显示其他社区作为背景。
- 不显示“社区简介”抽屉，也不提供社区信息入口。
- 保留一个很轻的当前位置导航：当前社区名 + 回全图。社区名只说明“我在哪”，不能打开社区简介。
- 搜索框保留在原工具栏位置，但只搜索当前社区内节点；它是社区内找节点的工具，不是全库搜索入口。
- 类型筛选保留在原工具栏位置，但只影响当前社区节点和关系；它是社区内降噪工具，不是全局筛选入口。
- 搜索尊重当前类型筛选：先筛选，再搜索，只在当前可见节点里找。
- 图例保留，但只解释当前社区里可见的节点、关系和状态含义；它不是全局社区列表，也不是社区切换入口。
- 首版交互目标是桌面鼠标阅读；#103 已补移动端 tap、Escape 和文本控件保护，完整键盘巡航体验如需增强再单独排期。
- 如果是从社区摘要进入，进入后关闭右抽屉，让画布成为主角；需要再看社区摘要时，回到全局选择阶段。
- 如果是用户明确点“打开详情 / 阅读某个节点”，可以进入社区并直接选中该节点，打开节点内容。
- 进入社区后，右侧区域不承载社区摘要，但用户点击节点时可以打开节点内容抽屉。

用户进入社区后的第一件事应该是：看节点、看线、hover 关系、点节点读内容。

### 3.3 社区视角不是全局放大

社区阅读模式不是简单把全局图放大，也不是带周边社区背景的局部镜头。

推荐形态：

- 一个新的 Sigma 社区空间。
- 只包含当前社区节点和内部边。
- 节点延续全局基础位置和相对关系，避免“换了一张图”。
- 节点延续全局社区身份色，不因为被选中、hover 或进入社区而换成统一状态色。
- 相机会把当前社区居中，给节点和标签足够阅读空间。
- 视觉语言比全局更“细读”：边更清楚，标签更克制但更有用，hover 关系更强。

这样既保留连续感，又避免全局上下文继续挤压社区内部阅读。

### 3.4 社区颜色是身份，不是状态

社区颜色承担“这是谁”的身份识别，必须跨全局、选中高亮、进入社区三段保持稳定。

规则：

- 全局默认状态下，每个社区使用自己的社区色。
- 全局选中某个社区后，该社区节点仍保持原社区色，不统一染成红棕色或其他状态色。
- 进入社区阅读模式后，节点仍保持同一社区色。
- hover、单选、多选、搜索命中、来源社区高亮等交互状态，只通过光晕、描边、透明度、标签清晰度、节点轻微放大和背景细节退后来表达。
- 强调色可以用于按钮、标签底、描边或光晕，但不能覆盖节点填充色。

原因：如果点中社区时节点先变色，进入社区时又切视角，用户会经历两次跳变。保持社区色稳定，后续做“社区被镜头拉近”的过渡动画才会自然。

与旧视觉对齐稿的关系：

- 早期稿里“选中态映射为节点状态色”“进入社区后部分节点改类型色”的判断，到本稿为止不再作为迁移依据。
- 新规则以“社区身份色稳定”为准：状态只做外层效果，不能覆盖节点填充色。

## 4. 功能迁移清单

| DOM 社区能力 | Sigma 迁移判断 | 方案 |
|---|---|---|
| 节点 hover 后高亮一度 / 二度 / 无关节点 | 可迁 | 复用现有关系聚焦规则，Sigma 监听节点进入/离开 |
| 直接连线变清楚、二度线弱化、无关线接近隐藏 | 可迁，首版保留 | 默认保留社区内部关系结构，但背景边低存在感；hover 或选中节点时，直接关系最清楚 |
| 节点光晕 / 选中 halo | 可迁 | 普通高亮走 Sigma，精致标签和光晕可用轻 DOM 覆盖层 |
| 标签预算、核心标签、hover 一度标签 | 可迁，首版保留 | 常驻标签只给核心、选中和搜索命中节点；hover 时临时显示当前节点和预算内一度邻居标签，覆盖层负责更稳定的标签表现 |
| 选中节点固定关系焦点，hover 临时覆盖，离开后恢复 | 可迁，首版保留 | 点击节点打开内容后，该节点同时成为稳定关系焦点；Sigma 内部维护 hover 节点，hover 临时覆盖，离开后恢复到当前单选节点 |
| 点击节点打开内容 | 可迁，首版保留 | 沿用现有节点内容抽屉命令，不和渲染器绑定；社区摘要抽屉关闭后，右侧仍允许节点内容抽屉出现；内容阅读态和画布关系焦点保持同步；内容已打开时再点当前节点不关闭，内容已收起但焦点仍在时再点当前节点重新打开内容 |
| 社区内多选节点 | 已由 #102 补齐 | Shift+点击精确增删当前社区内节点，打开选区抽屉，不扩大成整个社区 |
| 选择邻居 | 可迁，后续补 | 首版不放进社区阅读；后续随全局节点摘要能力补齐 |
| 选区提问动作 | 可迁，随多选切片补 | 多选后的总结、找缺口、生成主题页、探索关系等动作仍由抽屉承接 |
| 边类型颜色 | 可迁，首版保留 | 复用现有关系类型颜色，帮助用户读关系类别 |
| 边虚实 / 置信度 | 已由 #103 补齐 | 用轻量强弱变化表达，不盖过第一层关系颜色 |
| 边 hover / click 提示 | 已由 #103 补齐 | 只在社区模式开启边提示或边动作，避免拖慢全局大图 |
| 边显示调节控件 | 可迁，首版保留 | 只作用于当前社区阅读，用来给当前社区内部边降噪；不写入全局图谱显示状态，回全图时清空 |
| 搜索、类型筛选 | 可迁，首版保留 | 搜索框和类型筛选都保留原工具栏位置；社区阅读首版只搜索和筛选当前社区范围，不做全库搜索，也不影响全局筛选语义；回全图时清空 |
| 搜索结果定位 / 激活 | 可迁，首版保留 | 结果列表只包含当前社区节点；输入搜索词只更新结果和命中高亮，即使只有一个结果也不自动打开；点击搜索结果才打开该节点内容，并把它设为当前关系焦点；点击后保留搜索词、结果列表和命中高亮，便于继续比较；类型筛选变化时立刻按当前可见节点刷新结果列表和命中高亮；无结果时提示当前社区未找到，需要全库搜索则回全图 |
| 类型筛选隐藏当前阅读节点 | 可迁，首版保留 | 切换类型筛选后，如果当前阅读节点被筛掉，节点内容抽屉和当前关系焦点保留；画布提示“当前节点被筛选隐藏”，节点内容抽屉标题旁显示“已被筛选隐藏”，避免内容突然消失；该标题状态只是提示，不可点击，不负责清空筛选；用户仍可 hover 其它可见节点临时看关系，移开后回到隐藏提示；点击另一个可见节点时切换到新节点阅读；筛选改回让原节点重新可见时，只有它仍是当前阅读节点才自动恢复关系高亮，并清掉标题状态 |
| 鼠标拖动节点 | 可迁，首版保留 | 这是用户主动整理阅读空间，不等于系统自动重排；拖动后位置进入同一套图谱位置状态，回全图和下次进入社区都沿用，重置布局恢复当前知识库整张图的默认位置 |
| 固定/取消固定位置 | 可迁，后续补 | 这是节点摘要里的显式动作，不进入社区阅读首版；后续随全局节点摘要能力补齐 |
| 临时显示被筛选隐藏对象 | 可迁，后续补 | “显示这个对象 / 清除临时显示”后续继续工作，避免筛选或搜索把用户要看的节点藏死 |
| 社区核心节点列表 hover 预览 | 可迁 | 抽屉核心节点 hover 继续驱动画布 previewNode，不打开旧内容 hover 卡 |
| 返回全局并高亮来源社区 | 可迁 | 继续使用来源社区上下文；来源高亮只做定位提示，不自动打开社区摘要抽屉 |
| 工具栏：缩放、筛选、社区图例、重置布局、回全图 | 可迁 | 社区模式复用同一套控件，但图例只解释当前社区可见内容，重置布局恢复当前知识库整张图的默认位置，“回全图”语义是退出社区阅读 |
| 聚合 / 被收起对象提示 | 需要保留语义 | 若社区内仍存在聚合或被隐藏对象，抽屉要能说明原因并提供显示动作 |
| 抽屉关闭 / Escape 清理 | 可迁 | 关闭节点内容抽屉只收起内容，不清当前节点关系焦点；Escape 或空白表示退出当前节点阅读，要同时清当前节点焦点并关闭节点内容抽屉，但不清搜索词、结果列表和命中高亮；后续选区抽屉按多选规则单独处理 |
| 图谱数据刷新 / 更新动画 | 需要明确边界 | 刷新后社区仍存在则保持社区阅读；社区消失则回全局并提示对象不可用；当前阅读节点不再属于当前社区时清掉节点阅读，不把其他社区拉进来 |
| 键盘聚焦 / 可访问性 | 部分已由 #103 补齐 | 节点 hit target 和标签覆盖层承担 focus / blur、Escape 和文本控件保护；完整键盘巡航可单独增强 |
| 小地图 | 暂不作为首批重点 | Sigma 已有平移缩放；社区阅读更需要清楚关系，不需要再加一个小地图 |

### 4.1 遗漏排查补充

再次反查现有命令、抽屉和浏览器回归后，迁移设计需要覆盖这些能力：

- **社区内多选**：已由 #102 补齐。普通点击节点用于阅读；Shift+点击增删多选节点。多选必须是精确节点集合，不能扩大成整个社区。
- **邻居选择**：不能丢，但不进入社区阅读首版。后续随全局节点摘要能力补齐；若邻居包含社区外节点，抽屉保留完整清单，画布默认不把其他社区拉进来。
- **选区动作**：随社区多选切片保留。多选后的“总结这一组 / 找知识缺口 / 生成主题页 / 探索潜在联系”等动作仍由抽屉负责，和渲染器无关。
- **手动拖动节点**：必须保留。这和“不要自动重排节点”不是一回事；自动布局不能替用户移动节点，但用户自己拖动节点整理阅读空间仍然是社区阅读能力。拖动后的节点位置属于同一套图谱位置状态，返回全局后继续保留；用户点“重置布局”时恢复当前知识库整张图的默认位置。
- **节点摘要命令**：社区阅读首版不做节点摘要层；摘要层里的固定/取消固定位置、进入所属社区、选择邻居后续随全局节点摘要能力补齐。普通点击节点打开内容，不依赖节点摘要层。
- **临时显示对象**：不能丢，但不进入社区阅读首版。当节点因筛选、搜索或社区范围不可见时，“显示这个对象 / 清除临时显示”后续要有清楚路径。
- **核心节点列表 preview**：社区摘要或选区抽屉里的节点列表 hover 仍要能点亮画布节点，但不应恢复旧的内容 hover 卡。
- **工具栏与重置语义**：缩放、筛选、社区图例、边显示调节、重置布局继续存在；社区图例只解释当前社区可见内容，不带全局社区列表；边显示调节只影响当前社区，不影响全局图谱；“回全图”在社区阅读模式下表示退出社区，而不只是重置相机；“重置布局”恢复当前知识库整张图的默认节点位置。
- **抽屉关闭与 Escape**：关闭节点内容抽屉只收起内容，不退出社区，也不清当前节点关系焦点；Escape 或空白点击表示退出当前节点阅读，要同时清理当前节点焦点、关闭节点内容抽屉、清掉 hover 和临时显示，但保留搜索词、结果列表和命中高亮。
- **图谱更新**：知识库更新后如果当前社区仍存在，继续停留在社区阅读；如果社区消失，要回到全局并说明不可用；如果当前阅读节点消失或不再属于当前社区，要清掉节点内容和关系焦点，并给出轻提示，不能留在空内容或把其他社区节点拉进来。

这些能力都不要求增加用户步骤。它们只是保证迁到 Sigma 后，原先围绕社区视图的工作流不缩水。

## 5. 架构设计

### 5.1 一套 Sigma 主画布，两种模式

引擎层引入 Sigma 的两个模式：

- `global`：全局总览。
- `community-reading`：社区阅读。

首轮不要求大规模改名。可以先保留现有 Sigma 路线文件名，但行为上让它接受社区 focus；等稳定后再把命名从 `sigma-global` 收敛成更中性的 Sigma route。

关键变化是：Sigma 路线不能再强行忽略社区 focus。当前社区 focus 已经能让共享模型输出“只显示该社区”的节点和边，这个能力要交给 Sigma 消费。

### 5.2 社区规则仍由共享模型负责

社区阅读规则继续放在图谱引擎共享层，而不是塞进 Sigma 渲染器：

- 哪些节点出现。
- 哪些边出现。
- 哪些节点是核心 / 相关 / 外围。
- 哪些标签默认显示。
- 哪些边是骨架 / 相关 / 背景。
- 社区包围盒和居中目标。

Sigma 只负责把这些规则画出来，并处理相机、hover、拖动、点击。

### 5.3 关系聚焦作为 Sigma 社区内部状态

社区阅读首版下，关系焦点来源按优先级排列：

1. 鼠标 hover 的节点。
2. 当前单选节点，也就是用户最近点击阅读的节点。

这意味着点击节点不是只打开内容抽屉，也会把画布关系固定到这个节点。用户移动到其他节点时，可以临时查看另一组关系；鼠标离开后，关系焦点恢复到当前单选节点。关闭节点内容抽屉后，当前单选节点和关系焦点都保留；如果内容抽屉已经打开，再次点击当前单选节点不关闭内容，也不清掉焦点，避免误触打断阅读；如果内容抽屉已被收起，再次点击当前单选节点则重新打开内容。点空白或按 Escape 时，同时关闭节点内容抽屉并清掉当前节点焦点；如果当前有搜索词，搜索结果和命中高亮继续保留。点击另一个节点时，焦点和右侧内容切到新节点。画布不冻结，用户仍可继续拖动、搜索和切换节点。

如果当前单选节点被类型筛选隐藏，关系焦点仍记录在该节点上，但画布不强行显示不可见节点的关系，只显示“当前节点被筛选隐藏”的提示；右侧节点内容抽屉标题旁同步显示“已被筛选隐藏”。这个标题状态只是说明，不是按钮，点击它不触发清空筛选或显示对象。用户 hover 其它可见节点时，hover 节点临时成为关系焦点；鼠标离开后，画布回到当前节点被筛选隐藏的提示状态。用户点击另一个可见节点时，节点内容抽屉和关系焦点切到新节点，隐藏提示和标题状态消失。筛选改回让原节点重新可见时，只有它仍是当前阅读节点，才自动恢复原节点关系高亮并清掉标题状态。

关系焦点变化后：

- focus 节点轻微放大并有 halo，但填充色仍保持社区身份色。
- 一度节点保持清晰。
- 二度节点降低存在感。
- 无关节点明显压低。
- 一度边最清楚，二度边弱化，无关边接近隐藏。

这要复刻 DOM 社区现有体验，不能只做“hover 节点变亮”。

社区多选态下，画布要同时表达两件事：

- 哪些节点属于当前选区。
- 当前关系焦点落在哪个节点上。

也就是说，多选节点都应有选中态，但关系高亮只围绕 hover 节点或最后操作节点展开，避免多选后整张图同时高亮成噪声。

### 5.4 覆盖层继续存在，但只做该做的事

Sigma 负责画大部分节点和边。轻 DOM 覆盖层继续保留，用于：

- 可访问的节点 hit target。
- 更稳定的社区阅读标签。
- hover / focus 的键盘入口。
- 必要时的精致光晕或边提示。

覆盖层不拥有业务判断，不重新算关系，不决定哪些节点重要。它只消费共享模型和 Sigma 当前相机投影。

### 5.5 DOM 社区先保留为回退

迁移第一阶段不立即删除 DOM 社区路线。

保留原因：

- Sigma 运行失败时可以回退。
- 迁移期间可以做截图对照。
- 离线 HTML / 旧测试可以逐步切换。

主路径切到 Sigma 社区阅读后，DOM 社区不再承载新功能。

## 6. 进入、返回和抽屉规则

### 6.1 从全局进入社区

流程：

0. 仅真实社区可以进入；未分组不显示“进入社区”入口。
1. 用户在全局点社区。
2. 全局高亮该社区，右抽屉显示社区摘要。
3. 用户点“进入社区”。
4. 记录来源社区。
5. Sigma 切到社区阅读模式。
6. 相机把当前社区居中。
7. 关闭社区摘要抽屉，社区阅读里不展示也不提供社区简介入口。
8. 显示轻量当前位置导航：当前社区名 + 回全图。

结果：用户看到的是社区内部节点和关系，而不是另一个“社区说明页”。

### 6.2 从节点阅读动作进入社区

如果用户在全局摘要里明确点“打开详情 / 阅读某个节点”：

- 进入该节点所属社区。
- 选中该节点。
- 打开节点内容抽屉。
- 该节点作为关系焦点。

这是例外，因为用户明确选择了内容阅读，而不是单纯进入社区看图。

### 6.3 社区内节点阅读与后续多选

社区阅读模式内：

- 普通点击节点：打开或切换节点内容，同时把该节点固定为当前关系焦点。
- 单选时以节点内容阅读为主。
- hover 其他节点时，只临时切换关系焦点；离开后恢复到当前单选节点。
- 右侧抽屉允许显示节点内容；这和“进入社区后关闭社区摘要抽屉”不冲突。
- 关闭节点内容抽屉后，当前节点焦点继续保留。
- 内容抽屉已打开时，再次点击当前单选节点，保持当前节点阅读，不关闭内容，也不清掉焦点。
- 内容抽屉已收起但当前节点焦点仍保留时，再次点击当前单选节点，重新打开该节点内容。
- 点空白或按 Escape 时，关闭节点内容抽屉并清除当前节点焦点；如果搜索词还在，搜索结果和命中高亮继续保留。
- 点击另一个节点时，节点内容抽屉和关系焦点一起切到新节点。
- Shift+点击多选已由 #102 补齐。

社区阅读首版不提供节点摘要层，也不把“选择邻居 / 固定位置 / 进入所属社区”等节点动作塞进内容抽屉。节点摘要更适合全局视图，后续作为全局节点摘要能力单独补齐。

社区多选已补齐后：

- Shift+点击节点：增删选区节点。
- 多选时打开选区抽屉，展示结构事实和可执行动作。
- 多选不默认显示“进入社区”，因为用户已经在社区内。
- 多选只基于用户点过的精确节点，不自动扩大成整个社区。

后续补“选择邻居”时，如果邻居包含社区外节点，选区抽屉保留完整节点清单，但画布默认不把其他社区拉进来；用户需要看社区外节点时，通过“回全图”或主动显示对象完成。

后续多选状态下，如果用户再次普通点击某个节点，则回到单节点阅读；如果用户按 Escape 或点空白，则清除当前选区。

### 6.4 返回全局

返回全局时：

- 退出社区阅读模式。
- 回到全局 Sigma。
- 保留来源社区高亮，让用户知道刚才从哪里回来；这个高亮只是定位提示，不是社区摘要入口。
- 不自动打开社区摘要抽屉；用户需要摘要时，再点击该社区或使用全局高亮态里的入口。
- 回到全局后，如果用户点空白，清掉来源社区高亮，回到普通全图。
- 清空社区阅读里的搜索词、类型筛选和边显示调节，避免把社区内降噪状态带回全局。
- 保留用户拖动后的节点位置；它和全局使用同一套图谱位置状态。
- “重置布局”恢复当前知识库整张图的默认节点位置，不区分这次拖动发生在全局还是社区阅读里，也不是只重置当前社区。
- 保留 pin 这类用户位置状态；多选时按选区规则保留用户当前选区。

如果当前存在多选，返回全局后这些节点仍应保持选中；如果用户点空白清除，来源社区高亮和选区一起清掉。

### 6.5 数据刷新和不可用对象

图谱数据刷新时：

- 当前社区仍存在：继续留在社区阅读模式，按新数据重绘当前社区。
- 当前社区不存在：退出社区阅读，回到全局，并让抽屉说明社区不可用。
- 当前阅读节点不存在：关闭该节点内容，清掉该节点关系焦点，给出“当前节点已不可用”的轻提示。
- 当前阅读节点仍存在，但不再属于当前社区：关闭该节点内容，清掉该节点关系焦点，给出“当前节点已不在当前社区”的轻提示；社区阅读画布仍只显示当前社区，不把节点所属的新社区拉进来。
- 如果是多选，只移除不存在或已离开当前社区的节点；如果选区被清空，关闭选区抽屉并留在当前社区阅读模式。
- 当前临时显示对象不存在：清掉临时显示状态。
- 更新动画不应把用户从社区阅读模式强行弹走；但社区模式下可以比全局更克制地播放动画，避免干扰阅读。

这里和“类型筛选隐藏当前阅读节点”不同：筛选隐藏只是用户临时降噪，所以保留内容；数据刷新导致节点不存在或离开当前社区，是对象关系真的变了，所以不保留旧内容和旧关系焦点。

## 7. 视觉与交互标准

### 7.1 画面落点

进入社区后必须满足：

- 社区节点在画布内居中。
- 核心节点首屏可见。
- 不出现明显偏到边角的问题。
- 不因为标签变大导致节点关系读不清。
- 节点相对位置延续全局基础布局，系统不自动大幅重排。
- 节点社区身份色延续全局，不因选中或进入社区而换色。
- 用户可以手动拖动节点整理局部阅读空间；这不是系统自动重排，也不能因为保持连续感而被删除。拖动后的位置回到全局也保留，直到用户重置布局。

连续感的判断标准是：进入社区时动的是相机，不是系统替用户移动节点。节点可以随镜头被放大、居中，但节点之间的左右上下关系、社区整体形状和社区身份色都不能突然跳变。进入后，用户主动拖动节点属于整理阅读空间，不破坏这个原则。

如果社区很密：

1. 先减少普通节点标签。
2. 再降低背景边存在感。
3. 再轻微扩大相机视野。
4. 不默认用大幅重新布局解决拥挤。
5. 保留用户手动拖动节点，作为用户自己整理局部空间的出口。

首轮验收必须用“AI Agent 对比分析”这类真实密集社区检查，不能只用小样例。密集社区至少要满足：核心节点能第一眼看到、直接关系能读出来、标签不压住主要关系线。

### 7.2 标签

社区阅读模式的标签策略：

- 常驻标签只给核心节点、选中节点、搜索命中节点。
- 普通节点默认不常驻显示标签，避免社区一进入就变成文字墙。
- 桌面常驻标签大致控制在 6-10 个，窄屏控制在 3-6 个；如果核心、选中和搜索命中已经超过预算，就按优先级裁掉低优先标签。
- hover 节点必须显示标签。
- hover 时临时显示一度邻居标签，但仍受预算控制，避免一 hover 就出现标签墙。
- hover 离开后，临时邻居标签消失，画面回到常驻标签状态。
- 长标题要截断，不能遮住大面积关系线。

全局社区高亮态的标签策略：

- 当前社区节点标签按当前高亮规则显示。
- 未选中社区的社区级标签保留，用低透明度显示，作为可点击的地图路标。
- 未选中社区的节点级标签默认隐藏，避免和当前社区抢阅读焦点。
- 用户点击未选中社区标签或社区区域时，直接切换高亮社区并播放相机过渡，不要求先回全图；标签和区域都应该是有效目标。
- 这次点击只切换全局高亮对象，并把右侧社区摘要同步到新社区，不直接进入社区阅读模式；进入社区仍需要用户明确点击“进入社区”。
- 如果摘要抽屉当前关闭，这次用户主动点击其他社区可以打开新社区摘要；这不等于“回全图”时自动弹摘要。

### 7.3 颜色与状态

社区阅读模式的颜色策略：

- 节点填充色优先表达社区身份。
- 选中社区不改变节点填充色，只让该社区更清楚，让其他社区的节点细节和内部关系更淡。
- 当前节点焦点不改变节点填充色，只加光晕、描边、标签底或轻微放大。
- 后续多选节点不改变节点填充色，只用统一的选中外层状态表达“这些节点被选中”。
- 搜索命中不改变节点填充色，只用外圈、标签强调或标记表达。
- 进入社区后不把节点改成类型色或状态色；类型信息可以在图例、标签、hover 提示或抽屉中表达。

红棕色这类强调色可以继续存在，但定位是状态外层效果，不是节点身份色。

### 7.4 边

社区阅读模式的边比全局更清楚：

- 默认保留社区内部关系的结构感，不能进入后只剩孤立节点等待用户 hover。
- 骨架边和重要边保持可读，背景边低透明度、细线、低层级显示，不和直接关系抢前景。
- 边特别密时，先继续压低背景边，而不是把所有内部边同等画满。
- 首版用颜色表达关系类型。
- 置信度表达已由 #103 补齐，用轻量强弱变化，不额外增加抢眼纹理。
- hover 节点时，直接关系最清楚，二度关系弱化，无关关系接近隐藏。
- 已选中节点没有 hover 时，直接关系也应维持清楚，让用户读完内容后还能看见该节点关系。
- 边 hover / click 提示只在社区模式开启；全局大图不承担这类边交互。
- 被类型筛选隐藏的边不能因为 hover 又被唤醒。

全局社区高亮态的边策略：

- 当前社区内部边可以保留少量骨架关系，帮助用户判断是否进入。
- 未选中社区的内部边默认隐藏或接近不可见，不能形成白色关系网。
- 跨社区边只保留和当前社区直接相关的少量桥接提示，并且要明显弱于当前社区内部关系。
- 未选中社区的白色节点和白色连线不能成为视觉前景；它们只允许承担方位背景，不承担阅读信息。

### 7.5 控件与抽屉

社区阅读模式保留图谱日常控件，但语义要跟当前层级一致：

- 缩放按钮和滚轮缩放继续控制当前社区画布。
- 重置布局恢复当前知识库整张图的默认节点位置；如果用户在社区里拖动过节点，回全图也保留拖动结果，只有重置布局才清掉，且不是只清当前社区。
- 搜索框保留原工具栏位置，避免用户进入社区后重新找入口；搜索只作用于当前社区节点，没有结果时提示当前社区未找到，不展示社区外结果。
- 输入搜索词只更新当前社区结果列表和命中高亮；即使只有一个结果，也不自动打开节点内容，避免输入过程打断用户。
- 点击社区内搜索结果时，直接打开该节点内容，并把它设为当前关系焦点；这和普通点击节点阅读是同一个用户意图，不再要求用户到画布上多点一次。
- 点击搜索结果后，搜索词、结果列表和命中高亮继续保留；只有用户主动清空搜索或回全图时才清掉。
- 用户主动清空搜索时，只清搜索词、结果列表和命中高亮；如果当前正在阅读节点，节点内容抽屉和当前关系焦点继续保留。
- 类型筛选保留原工具栏位置，避免用户进入社区后重新找入口；它只影响当前社区可见节点和边，被筛掉的节点和边不能被 hover、关系高亮或搜索命中重新显示。
- 如果当前有搜索词，类型筛选变化后，搜索结果列表和命中高亮立刻按新的当前可见节点集合刷新；旧结果里被筛掉的节点不保留，也不标成旧结果。
- 如果类型筛选把当前正在阅读的节点筛掉，右侧节点内容和当前关系焦点继续保留；画布显示“当前节点被筛选隐藏”，节点内容抽屉标题旁显示“已被筛选隐藏”，让用户知道为什么节点不在画布可见集合里。标题状态不可点击，不清空筛选，也不显示对象。此时 hover 其它可见节点仍然临时显示这些节点的关系，移开后回到隐藏提示。点击另一个可见节点时，直接切换到新节点阅读，并清掉隐藏提示和标题状态。筛选改回让原节点重新可见时，只有原节点仍是当前阅读节点，才自动恢复它在画布上的关系高亮，并清掉标题状态。
- 搜索结果必须尊重当前类型筛选；如果关键词只命中被筛掉的节点，仍按当前可见范围无结果处理。
- 搜索词、类型筛选和边显示调节都是社区阅读里的局部降噪状态；回全图时清空，回到普通全图。
- 图例继续保留，但降级为当前社区的轻量读图说明：只解释当前可见节点类型、关系类型和交互状态，不展示全局社区列表，也不作为社区切换入口。
- “回全图”退出社区阅读，回到全局 Sigma，并保留来源社区高亮作为定位提示，但不自动打开社区摘要抽屉。
- 当前社区名作为轻量当前位置显示，和“回全图”放在同一导航区域；它不打开社区简介，不进入右抽屉，也不占用节点阅读画布。
- 边显示调节继续有效，但只作用于当前社区阅读；它不改变全局图谱边显示设置，也不能让全局大图承担社区级边事件开销。
- 右抽屉在社区进入后不展示社区简介，也不保留社区信息入口；只有用户点击节点打开节点内容，或后续多选节点时才出现。
- 关闭节点内容抽屉时，画布留在社区阅读模式，当前节点关系焦点继续保留。后续补选区抽屉后同样不退出社区。只有“回全图”或明确退出才离开社区。

## 8. 分阶段落地

阶段 A 只是内部骨架，不建议单独作为用户可验收版本。第一次给用户体验验收，至少要包含阶段 A + 阶段 B：只做到“能进社区但 hover 关系、边线层次、标签控制还没补齐”，仍然会像半成品。

### 阶段 A：Sigma 社区阅读骨架

- Sigma 路线接受社区 focus。
- 进入社区后只显示当前社区节点和内部边。
- 相机居中社区。
- 社区节点沿用全局基础位置和社区身份色，进入时不自动重排。
- 用户仍可用鼠标拖动节点整理局部阅读空间；拖动后回全图继续保留位置。
- 重置布局可以恢复当前知识库整张图的默认节点位置。
- 不显示社区简介抽屉，也不提供社区信息入口。
- 显示轻量当前位置导航：当前社区名 + 回全图。
- 搜索框保留原工具栏位置；搜索只查当前社区节点，命中后定位并高亮；唯一结果不自动打开；点击搜索结果直接打开节点内容并设为当前关系焦点，并保留搜索词和结果；主动清空搜索不关闭当前节点内容，也不清当前关系焦点；无结果时不展示社区外结果。
- 类型筛选保留原工具栏位置，只作用于当前社区，筛掉的节点和边不参与 hover 和关系高亮。
- 类型筛选变化后，搜索结果列表和命中高亮立即刷新为当前可见范围内的结果。
- 类型筛选把当前阅读节点筛掉时，节点内容和当前关系焦点保留，画布提示当前节点被筛选隐藏，节点内容抽屉标题旁显示“已被筛选隐藏”；标题状态不可点击；仍可 hover 其它可见节点临时看关系，移开后回到隐藏提示；点击另一个可见节点时切换到新节点阅读；筛选改回后，如果原节点仍是当前阅读节点，才自动恢复该节点关系高亮并清掉标题状态。
- 图例只解释当前社区可见节点、关系和状态，不出现全局社区列表或切换入口。
- 边显示调节只作用于当前社区内部边，不影响全局图谱。
- 普通点击节点打开内容，并固定该节点的关系焦点；内容开着时再点当前节点不关闭内容也不清焦点；关闭内容抽屉后焦点仍保留，再点当前节点会重新打开内容；hover 其他节点临时切换，离开后恢复。
- 空白和 Escape 会关闭节点内容抽屉，并清掉当前节点焦点；搜索词、结果列表和命中高亮不因此清掉。
- 图谱刷新后，社区仍存在则继续阅读；社区不存在则回全图并提示不可用；当前阅读节点不存在或离开当前社区时关闭节点内容并清掉关系焦点。
- 返回全局保留来源社区高亮作为定位提示，不自动打开社区摘要抽屉；点空白清掉来源高亮。
- 返回全局时清空社区搜索词、类型筛选和边显示调节，回到普通全图。
- DOM 社区仍可作为回退。

### 阶段 B：补齐 DOM 社区核心交互

- 节点 hover 关系聚焦。
- 节点 halo 和一度 / 二度 / 无关层级，但不改节点身份色。
- 边明暗和粗细层级：默认保留内部结构感，背景边退后，hover/选中直接关系最清楚。
- 边关系类型颜色。
- 常驻标签预算、hover 节点标签和一度邻居临时标签。
- 选中节点关系焦点，hover 临时覆盖，离开恢复。
- 点击节点打开内容后，该节点作为稳定关系焦点；内容抽屉和画布焦点不能脱节。

### 阶段 C：补齐多选、边、移动端和可访问性

- 社区模式开启边 hover / click。
- 补边提示和置信度表达。
- 覆盖层补键盘 focus / blur。
- 移动端 tap 行为稳定：tap 节点即选中并显示关系焦点，不依赖 hover。
- 补齐社区内 Shift+点击精确多选、选区抽屉和选区动作。
- 多选态下选中节点集合与当前关系焦点同时可辨认。
- 补齐节点列表 hover preview、临时显示对象等抽屉命令；邻居选择、固定/取消固定随全局节点摘要能力补齐。

### 阶段 D：收尾和债务清理

- 更新 ADR-26，把“社区不能迁 Sigma”的红线改成“已满足条件，进入迁移”。
- 浏览器回归稳定后，把 DOM 社区从主路径降为 fallback。
- 数据刷新、社区消失、节点消失、节点离开当前社区等异常路径通过后，再考虑删除 DOM 社区主入口。
- 后续单独评估是否删除 DOM 社区实现。

### 8.1 后续必须转 issue 的清单

这些不是社区阅读首版验收项，但不能被忘掉。转 PRD 和 issue 时必须单独列出：

- 全局节点摘要：在全局视图补节点摘要层，而不是塞进社区阅读首版。
- 社区内多选：已由 #102 补齐，Shift+点击精确增删节点，打开选区抽屉，保留选区动作。
- 边置信度虚实：已由 #103 补齐，用轻量强弱变化表达。
- 边 hover / click：已由 #103 补齐，只在社区模式开启边提示或边动作。
- 节点摘要动作：选择邻居、固定/取消固定位置、进入所属社区。
- 临时显示对象：被筛选、搜索或社区范围藏住的对象，要能临时显示并清除。
- 移动端和键盘可访问性：tap、Escape、文本控件保护已由 #103 补齐；完整键盘巡航体验可后续增强。
- DOM 社区债务：Sigma 社区阅读稳定后，再决定旧 DOM 社区入口和实现是否删除。

2026-07-05 状态补充：社区内多选已由 #102 补齐；边置信度表达、边 hover/click、移动端 tap、键盘/Escape/文本控件保护，以及 DOM 社区降为 fallback 的收尾已纳入 #103。剩余的全局节点摘要动作、固定/取消固定和临时显示对象，仍应作为独立后续能力处理。

防遗忘规则：从本设计拆 PRD 或 issue 时，上面每一项都要变成独立子 issue，父 issue 里保留 checklist 链接；这些子 issue 没创建前，不把父 issue 标记为完成。这样后续即使暂时不做，也不会靠记忆保存。

## 9. 不做什么

- 不做“进入社区后还带周边社区背景”的全局放大视角。
- 不增加用户操作步骤。
- 不显示社区简介抽屉。
- 不在社区阅读模式保留社区信息入口；社区摘要只留在全局选择阶段。
- 不把当前社区名做成社区简介入口；它只是当前位置提示。
- 不把移动端 tap、键盘 focus 和无 hover 设备体验放进首版验收。
- 不把社区阅读做成另一个新的第三套图谱。
- 不在首批重做右抽屉视觉。
- 不立即删除 DOM 社区实现。
- 不把 DOM 社区回退暴露成正常入口；它只是迁移期保险。
- 不为了拥挤而默认重排成另一张图。
- 不用统一状态色覆盖社区节点填充色。
- 不隐藏未选中社区的社区级标签；它们是全局高亮态里的切换入口。

## 10. 验收标准

首版用户体验验收：

- 从全局进入社区后，用户感觉是在同一套图谱系统里深入，而不是换到另一个图谱。
- 社区节点默认居中，核心节点可见。
- 社区节点在全局、选中高亮、进入社区后保持同一社区身份色。
- 全局高亮态下，未选中社区的社区级标签和社区区域都可点击切换；切换后右侧摘要同步到新社区，摘要抽屉若关闭则打开新社区摘要；仍停留在全局高亮态，不自动进入社区阅读；未选中社区的节点级标签和内部白线网不抢前景。
- 进入社区后没有社区简介抽屉和社区信息入口挡住画布。
- 社区阅读里能看到轻量当前位置：当前社区名 + 回全图；社区名不是社区简介入口。
- 社区阅读里保留轻量图例，只解释当前社区可见节点、关系和状态含义；不出现全局社区列表，不提供社区切换入口。
- 进入社区时系统不自动重排节点；用户手动拖动节点能力保留。
- 用户拖动节点后，回全图和下次进入社区都保留该位置；点击重置布局恢复当前知识库整张图的默认位置。
- 搜索框保留在原工具栏位置，但搜索范围只限当前社区；不会切到全库搜索。
- 类型筛选保留在原工具栏位置，但只筛当前社区节点和关系；不会沿用成全局筛选。
- #103 已补移动端 tap、键盘/Escape/文本控件保护；完整无 hover 设备体验继续随真实使用校准。
- 默认常驻标签只显示核心节点、选中节点和搜索命中节点；不能把当前社区所有节点标签一次性铺开。
- hover 节点能清楚看到直接关系和相关节点。
- hover 时临时显示当前节点和预算内一度邻居标签，离开后回到默认常驻标签状态。
- 边 hover / click 已由 #103 补齐；边线层次仍主要由节点 hover、选中节点和筛选状态驱动。
- 默认能看出当前社区内部关系结构，但背景边不能糊成一张线网。
- hover 或选中节点时，直接关系必须明显强于二度关系和背景关系。
- 边保留关系类型颜色；置信度用轻量强弱变化表达，不盖过类型颜色。
- 点击节点能打开内容，并把该节点作为稳定关系焦点；内容开着时再点当前节点不关闭内容也不清焦点；关闭内容抽屉后焦点仍保留，再点当前节点会重新打开内容；hover 其他节点临时覆盖，离开后恢复到当前单选节点。
- 节点内容抽屉允许在右侧打开；社区摘要抽屉不允许在社区阅读里默认保留。
- 搜索只在当前社区内找节点，命中后定位并高亮该节点；唯一结果不自动打开；点击搜索结果直接打开节点内容并设为当前关系焦点，并保留搜索词和结果；主动清空搜索不关闭当前节点内容，也不清当前关系焦点；搜不到时不展示社区外结果。
- 类型筛选只在当前社区内生效；筛掉的节点和边不会因为 hover 或关系高亮重新出现。
- 类型筛选变化后，当前搜索结果列表和命中高亮立即按筛选后的可见节点重算，不保留被筛掉的旧结果。
- 如果类型筛选筛掉当前阅读节点，节点内容和当前关系焦点仍保留，画布提示当前节点被筛选隐藏，节点内容抽屉标题旁显示“已被筛选隐藏”；标题状态不可点击；仍可 hover 其它可见节点临时看关系，移开后回到隐藏提示；点击另一个可见节点时切换到新节点阅读；筛选改回后，如果原节点仍是当前阅读节点，才自动恢复该节点关系高亮并清掉标题状态。
- 搜索尊重类型筛选，只搜索当前筛选后的可见节点；被筛掉的节点不会被搜索临时唤回。
- 边显示调节只改变当前社区内部边的显示，不改变全局图谱；返回全图后该调节被清空。
- 社区阅读首版不出现多选入口，不出现节点摘要层；社区多选、选择邻居、固定/取消固定等能力作为后续 issue 进入队列，不在首版里假装完成。这里不包括鼠标拖动节点，拖动能力要保留。
- 关闭节点内容抽屉不会退出社区阅读，也不会清掉当前节点焦点；Escape/空白会关闭节点内容抽屉并清掉当前节点焦点，但保留搜索词、结果列表和命中高亮。
- 图谱刷新后，社区存在则继续阅读；社区消失则回到全图并给出可理解提示；当前阅读节点消失或离开当前社区时，关闭节点内容并清掉关系焦点，不把其他社区节点拉进当前社区阅读。
- 返回全局后，来源社区仍然高亮，但这只是定位提示，不自动打开社区摘要抽屉。
- 返回全局后，社区阅读里的搜索词、类型筛选和边显示调节被清空，用户看到普通全图。
- 返回全局后点空白会清掉来源社区高亮，回到普通全图。
- 首次可体验版本至少完成社区进入、居中、当前位置导航、轻量社区图例、无社区简介入口、社区内搜索、当前社区类型筛选、hover 关系聚焦、默认边线分层、常驻标签预算和 hover 临时标签；只完成骨架不算体验达标。

完整迁移补充验收：

- Shift+点击能在社区内精确多选节点，打开选区抽屉，不扩大成整个社区。
- 多选后仍能看到哪些节点被选中，也能看到当前 hover 的关系焦点。
- 边置信度表达已由 #103 补齐，用轻量强弱变化避免盖过第一层关系颜色。
- 边 hover / click 提示已由 #103 补齐，只在社区模式开启。
- 被筛选、搜索或社区范围藏住的对象，可以通过后续补齐的临时显示能力看见并清除。

工程验收：

- Sigma 社区模式复用共享社区规则，不复制 DOM 的判断逻辑。
- Sigma 社区模式复用全局社区色规则，交互状态不能覆盖节点身份色。
- 现有 DOM 社区浏览器回归里的首版关键行为，在 Sigma 社区模式下有等价覆盖：进入社区、居中且不自动重排、手动拖动节点且回全图保留位置、重置布局恢复当前知识库整张图的默认位置、当前位置导航、轻量社区图例、无社区简介入口、hover 关系、原位置社区内搜索、原位置社区内类型筛选、社区内边显示调节、点击读内容、刷新异常降级、返回全局并保留来源社区高亮、清空社区搜索/筛选/边显示调节。
- 社区内多选、边置信度虚实、边 hover/click、移动端 tap、键盘/Escape/文本控件保护和 DOM 社区收尾已由 #102-#103 补齐；邻居选择、固定位置、临时显示对象仍进入后续 issue 队列，不在当前社区阅读收尾里假装完成。
- 全局 Sigma 性能不因社区边事件和 hover 逻辑变差；边事件只在社区阅读模式开启。
- Sigma 不可用时仍有回退路径。
- 旧“过渡动画”设计不再作为当前实施依据。

## 11. 测试策略

### 11.1 单测

- Sigma 路线在社区 focus 下不再忽略 focus。
- 社区 focus 下 adapter 只输出当前社区节点和内部边。
- 社区 focus 下节点颜色沿用社区身份色，选中/hover/搜索命中只输出外层状态。
- 进入社区时使用全局基础位置，不触发自动重排；手动拖动时节点会跟随移动，并写入同一套图谱位置状态。
- 重置布局清空当前知识库整张图里用户拖动产生的位置状态，恢复默认布局。
- 标签计算默认只输出核心、选中和搜索命中节点；hover 时再输出当前节点和预算内一度邻居。
- 边样式计算默认输出骨架/重要/背景层级；hover 或选中节点时直接边强于二度边，二度边强于背景边。
- 关系聚焦规则在 Sigma 模式复用现有一度 / 二度 / 无关判断。
- 社区阅读模式下搜索范围只包含当前社区节点，不返回社区外节点。
- 社区搜索命中节点会进入搜索命中态；即使只有一个搜索结果，也不自动打开内容或改变当前节点选择；无结果时不改变画布选择。
- 点击社区搜索结果时，打开该节点内容，并把该节点设为当前关系焦点。
- 点击社区搜索结果后，搜索词、结果列表和搜索命中态保持不变。
- 主动清空社区搜索时，只清搜索词、结果列表和搜索命中态；当前节点内容和关系焦点不变。
- 社区阅读模式下类型筛选范围只包含当前社区节点和边，不影响全局视图状态。
- 被类型筛选隐藏的节点和边不会被 hover、关系高亮或搜索命中重新显示。
- 类型筛选变化时，如果存在搜索词，搜索结果列表、搜索命中态和命中标签都按筛选后的可见节点集合重算；被筛掉的旧结果不保留。
- 当前阅读节点被类型筛选隐藏时，不关闭节点内容，不清当前关系焦点，并输出“当前节点被筛选隐藏”的提示状态；节点内容抽屉标题旁输出“已被筛选隐藏”状态，且该状态不可点击；hover 其它可见节点时临时使用 hover 节点关系焦点，离开后回到隐藏提示；点击另一个可见节点时，当前节点内容和关系焦点切到新节点，隐藏提示和标题状态清除；原节点重新可见时，只有它仍是当前阅读节点才恢复原节点关系高亮并清掉标题状态。
- 搜索结果基于类型筛选后的可见节点集合；被筛掉的节点不能进入搜索结果。
- 退出社区阅读时清空社区搜索词、类型筛选和边显示调节，回到普通全图状态。
- 社区图例的数据只来自当前社区可见节点和边，不包含全局社区列表。
- 社区阅读模式下边显示调节只改变当前社区内部边，不修改全局图谱状态；退出社区时重置。
- 后续单测覆盖：Shift+点击在社区 focus 下输出精确 `nodes` 选区，不输出 `community` 选区。
- 后续单测覆盖：多选状态下最后操作节点可作为关系焦点 fallback。
- 点击节点打开内容后，关系焦点固定到该节点；内容开着时再点当前节点不关闭内容也不清焦点；关闭内容抽屉后焦点仍保留，再点当前节点会重新打开内容；Escape/空白会关闭内容抽屉并清掉焦点，但保留搜索词、结果列表和命中高亮；hover 其他节点只临时覆盖，离开后恢复到当前单选节点。
- 社区阅读首版不要求节点摘要命令；邻居选择、固定/取消固定、临时显示对象后续进入全局节点摘要或增强切片。
- 数据刷新后，当前社区不存在时退出社区阅读并回全图；当前阅读节点不存在或离开当前社区时，关闭节点内容、清掉关系焦点，并输出对应轻提示。
- 后续单测覆盖：临时对象不存在时，状态能被清理或降级。

### 11.2 浏览器回归

新增 Sigma 社区阅读回归，对齐现有 DOM 社区关键行为：

- 进入社区后社区节点居中。
- 进入社区后节点相对位置沿用全局基础布局，不自动重新排版。
- 用户可以拖动节点；拖动后该节点移动，关系焦点、hover 和点击读内容仍正常。
- 拖动后回全图，节点位置继续保留；再次进入社区也沿用该位置。
- 点击重置布局后，当前知识库整张图里拖动产生的位置被清掉，所有节点回到默认位置；这不是只重置当前社区。
- 搜索框仍在原工具栏位置；输入关键词只返回当前社区节点，不触发全库结果。
- 类型筛选仍在原工具栏位置；切换后只影响当前社区节点和边，不改全局筛选状态。
- 无社区简介抽屉，也无社区信息入口。
- 显示当前社区名和回全图入口；点击社区名不会打开社区简介。
- 图例只说明当前社区可见节点、关系和状态；不显示全局社区列表，也不能用图例切换社区。
- 默认常驻标签只出现核心节点、选中节点和搜索命中节点，不出现全社区标签墙。
- 全局选中社区后，社区节点仍保持原社区色，不统一变成状态色。
- 全局选中社区后，未选中社区的社区级标签仍可见且可点击切换。
- 点击未选中社区标签或社区区域后，直接切换全局高亮社区，右侧社区摘要同步到新社区；如果摘要抽屉关闭，则打开新社区摘要；不需要先点空白，也不会直接进入社区阅读。
- 全局选中社区后，未选中社区的节点级标签隐藏，内部白色关系网隐藏或接近不可见。
- 进入社区后，社区节点颜色与全局该社区一致。
- hover A 后 B/C 为一度，D 为二度，F 为无关。
- 直接边比二度边清楚，二度边比无关边清楚。
- 没有 hover 时仍能看出社区内部关系骨架，但背景边不会形成一整片抢眼线网。
- 点击节点后，该节点直接关系保持清楚；移动到其他节点 hover 时临时切换焦点，离开后恢复选中节点关系。
- 边保留关系类型颜色；置信度用轻量强弱变化表达，不盖过类型颜色。
- 边 hover / click 在社区模式内触发轻量提示，不打开完整抽屉。
- hover 不改变节点位置，系统也不因为 hover 自动重排节点。
- hover 节点显示自身标签，并只临时显示预算内一度邻居标签；离开后临时标签消失。
- 搜索当前社区内节点后定位并高亮命中节点；即使只有一个结果也不自动打开内容；点击搜索结果后打开该节点内容，并把该节点设为当前关系焦点，同时保留搜索词、结果列表和命中高亮。
- 主动清空搜索后，搜索词、结果列表和命中高亮消失；如果节点内容正在打开，它仍保持打开，当前节点关系焦点不变。
- 搜索社区外关键词时不展示社区外结果，只显示当前社区未找到。
- 切换类型筛选后，当前社区只保留匹配类型的节点和边。
- 切换类型筛选后，搜索结果列表和命中高亮立即按当前可见范围刷新，不保留被筛掉的旧搜索结果。
- 切换类型筛选后，如果当前阅读节点被隐藏，节点内容抽屉仍打开，当前节点关系焦点不变，并显示“当前节点被筛选隐藏”。
- 当前阅读节点被筛选隐藏时，节点内容抽屉标题旁显示“已被筛选隐藏”；这个状态不可点击，不清空或调整类型筛选。
- 当前阅读节点被筛选隐藏时，hover 其它可见节点仍能临时显示这些节点的关系；鼠标移开后回到“当前节点被筛选隐藏”状态。
- 当前阅读节点被筛选隐藏时，点击另一个可见节点会切换到新节点阅读，隐藏提示和标题状态消失。
- 再次切换类型筛选让原节点重新可见后，如果原节点仍是当前阅读节点，该节点关系高亮自动恢复，隐藏提示和标题状态消失；如果用户已经切换到新节点，不再恢复原节点高亮。
- 搜索被当前类型筛选隐藏的节点时，不显示该节点，也不临时显示该节点。
- hover 节点时，被类型筛选隐藏的节点和边仍然不可见。
- 调整边显示后，只影响当前社区内部边；返回全图后全局边显示不受影响，再次进入社区时使用默认状态。
- 点击节点后打开内容，关系焦点固定到该节点。
- 内容抽屉已打开时，再次点击当前节点，内容抽屉仍保持打开，当前节点关系焦点不变。
- 节点内容抽屉在右侧打开；关闭后仍留在社区阅读模式，并保留当前节点关系焦点。
- 节点内容抽屉被关闭但当前节点关系焦点仍保留时，再次点击当前节点会重新打开该节点内容。
- 点空白或按 Escape 后，节点内容抽屉关闭，当前节点关系焦点清掉，仍留在社区阅读模式；如果搜索词还在，搜索结果和命中高亮保留。
- 社区阅读首版不出现多选入口，不出现节点摘要层，也不出现“选择邻居 / 固定位置”等节点摘要动作。
- 后续回归覆盖：Shift+点击 A/B 后打开选区抽屉，标题为选中 2 个节点，不显示“进入社区”动作。
- 后续回归覆盖：多选后 hover C，只临时显示 C 的关系焦点，A/B 仍保留选中态。
- 后续回归覆盖：被筛选隐藏的节点可以通过“显示这个对象”临时显示，并能清除。
- 后续回归覆盖：关闭选区抽屉后仍留在社区阅读模式，Escape 或空白清除选择。
- 模拟图谱刷新：社区仍存在时不退出；社区不存在时回全图并展示不可用提示；当前阅读节点不存在时关闭内容并清掉关系焦点；当前阅读节点离开当前社区时不把它拉进画布。
- hover 其他节点临时覆盖，离开后恢复到当前单选节点。
- 返回全局后来源社区高亮只作为定位提示，社区摘要抽屉不自动弹出，社区搜索、筛选和边显示调节已清空；点空白后清掉来源社区高亮。

### 11.3 手动视觉验收

至少检查这些真实社区：

- “AI Agent 对比分析”这类节点较多社区。
- 小社区。
- 长标题节点多的社区。
- 边特别密的社区。
- 没有明显核心节点的社区。

每类都要看桌面和窄屏。

每次视觉验收至少留三张对照截图：

- 全局普通态。
- 全局选中当前社区态。
- 进入 Sigma 社区阅读后。

截图对比重点不是“有没有动画”，而是四件事：社区颜色是否连续、节点相对形状是否连续、当前社区是否居中且可读、未选中社区的标签是否像地图路标而不是和白色线网一起抢前景。

## 12. 成功信号

这次迁移成功，不是因为 Sigma 长得和 DOM 一模一样，而是因为：

- 用户进入社区后能立刻读关系。
- 全局和社区不再像两张图。
- 新增图谱能力不再需要先问“Sigma 一套、DOM 一套分别怎么做”。
- 旧 DOM 社区可以逐步退出主路径，而不是继续承载新功能。
