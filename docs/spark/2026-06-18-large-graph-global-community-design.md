# Large Graph Global/Community Interaction Design

日期：2026-06-18
状态：待用户复核
当前分支：`codex/graph-performance-optimization`
视觉稿：[`docs/spark/mockups/2026-06-18-large-graph-global-community-flow.html`](mockups/2026-06-18-large-graph-global-community-flow.html)

## 目的

这份设计确定 llm-wiki 图谱在 5000+ / 10000+ 节点规模下的产品交互方向。

目标不是把当前图谱做成一个更复杂的自动模式系统，也不是照搬 `graphify` 或 `nashsu/llm_wiki`。目标是明确一条长期路线：

**全局图谱负责看结构，社区聚焦负责读内容。**

当前图谱已经有社区色块、社区图例、筛选、搜索、右侧抽屉、Pin、社区聚焦和密度模式。后续性能优化应围绕这些既有用户动作收敛，而不是增加一套让用户难以理解的自动模式切换。

## 设计结论

采用“全局降级，社区聚焦升级”的双层体验。

1. 全局视角默认支持 5000+ / 10000+ 节点，但只显示地图级信息。
2. 普通节点点击只触发轻量查看，不改变全局视角。
3. 社区色块或社区图例点击先选中社区并显示社区摘要，用户再通过按钮进入社区聚焦。
4. 社区聚焦后恢复卡片、标签、关系细节和阅读抽屉。
5. 搜索和筛选不创建新模式，只改变当前全局或社区视角里的可见范围。
6. 缩放只调整细节密度，不自动切换全局/社区主模式。
7. 返回全图以可见按钮为主，双击空白只作为辅助快捷方式。

一句话规则：

**单击决定当前检查什么；按钮决定是否深入；缩放只决定当前画多少细节。**

## 为什么不做自动切换

不建议根据缩放自动在“全局大图”和“局部阅读”之间切换。

原因：

- 当前已有社区入口和筛选入口，用户意图已经足够明确。
- 自动切换容易让用户迷失：只是缩放一下，界面突然换成另一套阅读状态。
- 大图性能优化需要稳定的规则，不能让细节组件在缩放边界反复出现和消失。
- 现有社区色块点击已经是自然的社区检查入口，应继续强化它；真正深入社区由明确按钮承接。

允许的自动行为只有信息密度变化：

- 缩远时隐藏普通标签、弱边和复杂效果。
- 放近时显示更多关键标签。
- 手势进行中进一步降级，手势停止后补回允许显示的细节。

## 交互规则

### 1. 全局视角

全局视角是地图，不是阅读界面。

显示：

- 节点点位。
- 社区颜色。
- 少量关键节点标签。
- 重要边或社区间骨架关系。
- 搜索命中和选中节点高亮。
- 社区图例和筛选入口。

隐藏或降级：

- 普通节点卡片。
- 大段文本摘要。
- 全量节点标签。
- 全量边。
- 复杂阴影、动画和长时间运行的动态布局。

全局视角点击普通节点：

- 保持图谱视角不变。
- 高亮该节点和少量直接邻居。
- 右侧打开轻量查看抽屉。
- 不自动进入社区聚焦。
- 不自动打开完整阅读内容。

全局选中和抽屉优先级：

- 节点命中优先于社区色块。用户点到具体节点时，抽屉显示节点摘要。
- 点社区色块、社区图例或社区背景，但没有点中具体节点时，抽屉显示社区摘要。
- 悬停只做临时提示，不接管右侧抽屉。
- 搜索和筛选可以改变高亮范围，但不抢占已选中的节点或社区。
- Pin 是持久标记，不等于当前选中对象，也不自动切换抽屉。
- 单击全局空白处只清空当前选中对象，抽屉回到全局概览或搜索结果列表；不改变缩放，也不返回全图。

### 2. 全局轻量查看抽屉

当前代码里点击节点会打开右侧抽屉。这个方向保留，但抽屉需要重新定义为“轻量查看”，不要变成图上大卡片的替代品。

这里不是两个抽屉，而是一个抽屉、两种内容层级：

- 点社区色块或社区图例时，抽屉显示社区摘要。
- 点社区里的具体节点时，抽屉切到节点摘要。
- 如果社区和节点同时存在，节点是主对象，社区只做上下文。

轻量抽屉显示：

- 节点名称。
- 类型。
- 所属社区。
- 连接数。
- 3-5 个最重要相邻节点。
- 1-2 条最强关系。
- “进入所属社区”动作。
- “打开详情 / 阅读”动作，但全局图谱不重排、不出现图上卡片。
- 预留“向 Agent 询问这个节点”的扩展位，但不作为第一轮必做入口。

社区摘要时，这些字段会换成：

- 社区名称。
- 节点总量。
- 核心节点概览。
- 关键桥接关系。
- “进入社区”动作。
- “查看核心节点列表”动作。

轻量抽屉不显示：

- 长正文。
- 大量相邻节点列表。
- 复杂 Markdown 阅读内容。
- 会迫使全局图谱重排的大组件。

轻量抽屉的定位：

- 它是全局地图的检查面板。
- 它帮助用户决定是否深入。
- 它不承担完整阅读。
- 它的文案和按钮要随“社区”或“节点”这两个对象切换，而不是固定不变。
- 没有选中对象时，它可以显示全局概览、搜索结果或空状态，但不混进节点/社区摘要。

### 3. 社区入口

社区色块和社区图例点击继续作为“选择社区”的主入口。

这个设计是合理的，因为用户点击社区本身就是明确动作：我要先看这个社区是什么，再决定是否深入。

点击社区色块或社区图例后：

- 全局视角保持不变。
- 高亮该社区和它的少量跨社区关系。
- 右侧轻量抽屉显示社区摘要。
- 不立刻进入社区聚焦。
- 抽屉里的“进入社区”按钮才触发社区聚焦。
- “查看核心节点列表”只在抽屉内展开列表或高亮核心节点，不改变主图视角。

进入社区聚焦后：

- 图谱视角 fit 到该社区。
- 只显示该社区内的核心节点和必要关系。
- 其他社区退出主要视野或降为背景。
- 显示卡片、更多标签、阅读抽屉和社区摘要。

普通节点也可以进入社区，但必须通过明确动作：

- 在轻量抽屉里点“进入所属社区”。
- 不建议单击节点直接进入社区。
- 第一轮不把双击作为社区入口；后续只有在实测误触风险很低时再考虑加入。

### 4. 社区聚焦视角

社区聚焦是阅读和理解模式。

显示：

- 社区标题和返回全局入口。
- 社区核心节点卡片。
- 核心关系。
- 社区摘要。
- 关键节点列表。
- 与其他社区的桥接关系。
- 阅读抽屉或详情抽屉。
- 预留“向 Agent 询问这个社区”的扩展位，但不作为第一轮性能优化必做入口。

规模控制：

- 如果社区本身很大，默认只显示核心节点。
- 其余节点通过筛选、搜索、一跳邻居或“展开更多”进入。
- 不在社区聚焦里一次性恢复所有节点卡片。

### 5. 搜索和筛选

搜索和筛选是对当前视角的收窄，不是模式切换。

全局视角中：

- 搜索命中节点强高亮。
- 命中节点的少量相邻节点轻高亮。
- 非命中节点降淡。
- 如果没有选中节点或社区，右侧轻量抽屉可以显示搜索命中列表。
- 如果用户点中某个搜索结果，抽屉切到该节点摘要。
- 清空搜索后，保留当前选中对象；如果没有选中对象，则回到全局概览。

社区视角中：

- 搜索只在当前社区内优先匹配。
- 可提供“扩展到全局搜索”的动作。

筛选后如果剩余节点少，系统可以逐步恢复更多标签和边，但仍遵守当前视角的主规则。

### 6. 返回全图与双击

返回全图是导航动作，必须有可见入口，不依赖用户记住手势。

主规则：

- “回全图”按钮是主入口。
- 双击真正的图谱空白背景可以作为快捷回全图。
- 单击空白只取消当前选中，不改变当前视角。
- 在社区聚焦里，单击空白只清空当前节点/关系选择，仍留在当前社区。
- 从社区聚焦回全局，需要点“回全图”按钮，或双击真正的空白背景。

双击限制：

- 双击节点不返回全图，不进入社区。
- 双击社区色块不返回全图，不进入社区。
- 双击社区图例、搜索框、工具栏、右侧抽屉不触发图谱导航。
- 双击边不应作为回全图入口；边未来可以升级为关系检查对象。
- 现有“节点双击取消固定位置”的旧能力不应作为主交互继续强化。更好的方向是在节点轻量抽屉或节点工具里提供明确的固定/取消固定动作；旧双击行为最多作为过渡兼容。

键盘规则：

- Escape 先关闭搜索、工具栏或临时提示。
- 如果当前有选中节点/社区，Escape 清空选中对象。
- 如果已经在社区聚焦且没有选中对象，Escape 可以返回全图。
- Escape 不清除 Pin，不重置布局。

## 大规模降级规则

全局图谱的目标规模按 5000+ / 10000+ 设计。

这不意味着万级图谱要完整显示全部细节。参考 `graphify` 的实际策略，大规模流畅来自降级和聚合，而不是硬画所有内容。

### 节点

- 5000+：全局默认点位为主，只显示核心标签。
- 10000+：进一步减少标签，只保留搜索命中、选中、核心节点和社区代表节点。
- 节点大小按重要性变化，但普通节点不变成卡片。

### 边

- 默认只显示重要边、强关系边、社区间骨架边。
- 弱边默认隐藏或极淡。
- 拖动、缩放、平移过程中隐藏更多边。
- 停止交互后再补回允许显示的边。

### 标签

- 标签预算按当前可见节点数控制。
- 标签优先级：搜索命中 > 选中节点 > 社区代表节点 > 高连接节点 > 普通节点。
- 普通节点标签只能在悬停、轻量抽屉或社区聚焦中出现。

### 社区

- 社区是全局大图的主要组织单位。
- 当总节点超过阈值时，全局优先展示社区结构。
- 如果完整节点图会影响流畅度，可以提供社区聚合视图。
- 社区聚合视图中，社区本身是节点，社区之间的连接代表跨社区关系强度。
- 社区聚合是全局视角的一种渲染层级，不是新的产品模式；点击后仍然先进入社区摘要，再由按钮进入社区聚焦。

### 动画与物理布局

- 全局图谱不应持续运行高消耗动态布局。
- 初始布局稳定后应停下来。
- 拖动时只更新必要对象。
- 社区聚焦可以允许更丰富的局部动画，但仍要受节点数量限制。

## 技术方向

本设计不在这一阶段锁死具体库，但锁死技术主线：

**一个图谱数据底座，一个全局大图渲染器，一套社区/筛选/搜索规则，细节阅读由抽屉和社区聚焦承接。**

可评估的全局大图渲染方向：

- `Sigma + Graphology`：更偏现代 WebGL 图谱，适合桌面版长期路线。
- `vis-network`：`graphify` 使用的方向，Canvas 图谱成熟，聚合视图策略清晰。
- 当前 DOM/SVG：继续用于小图、社区阅读或离线细节模式，不再要求硬撑万级全局图。

明确不做：

- 不同时引入 `Sigma` 和 `vis-network` 两套全局渲染器。
- 不做三套坐标、三套点击、三套筛选。
- 不把卡片和长文本塞进万级全局画布。
- 不因为引入大图渲染器而丢掉现有阅读抽屉和社区体验。

未来桌面版方向：

- 工作台仍以 React/Vite 为基础。
- 桌面壳优先 Tauri。
- 大图布局计算可放到后台线程或桌面后端。
- 图谱数据、社区检测、搜索和阅读抽屉保持同一产品语义。

## 与参考项目的关系

### graphify

借鉴：

- 大图全局降级。
- 超大图社区聚合。
- 只给关键节点显示标签。
- 交互时隐藏部分边。
- 稳定后停止高消耗布局。

不照搬：

- 不把 llm-wiki 做成静态 HTML 图谱工具。
- 不把完整阅读体验交给图谱画布。
- 不让社区聚合替代社区阅读。

### nashsu/llm_wiki

借鉴：

- 桌面应用方向。
- 成熟图谱库承接大图。
- 布局计算后台化。
- 节点数越大，标签和边越克制。
- 图谱洞察面板的产品思路。

不照搬：

- 不放弃 llm-wiki 现有的社区聚焦、Pin、离线图谱和阅读抽屉。
- 不把图谱只做成结构浏览器。

### 当前 llm-wiki graph-engine

保留：

- 社区色块和社区图例。
- 社区聚焦。
- 筛选和搜索。
- Pin。
- 右侧抽屉。
- 离线 HTML 图谱能力。
- 现有图谱数据语义。

调整：

- 全局视角减少卡片化表达。
- 点击节点的右抽屉改成轻量查看。
- 社区聚焦作为细节阅读主入口。
- 大规模场景引入更强的降级和聚合规则。

## 产品状态机

核心状态只有四类：

1. `global`
   - 全局地图。
   - 支持筛选、搜索、全局概览、节点轻量查看、社区摘要。

2. `global + selected community`
   - 全局地图保持不变。
   - 右侧轻量抽屉显示社区摘要。
   - 可以查看核心节点列表或进入社区聚焦。

3. `global + selected node`
   - 全局地图保持不变。
   - 右侧轻量抽屉显示节点信息。
   - 可以进入所属社区或打开详情。

4. `community focus`
   - 聚焦社区。
   - 恢复卡片、核心关系、社区阅读抽屉。
   - 支持返回全局。

不新增“自动阅读模式”或“缩放触发模式”。

高级选区不是核心状态：

- Shift+点击、多选、邻居扩展可以继续存在，但它是高级分析选区。
- 高级选区也使用右侧抽屉显示摘要和动作，不改变全局或社区主模式。
- 如果它影响大图流畅度，优先降级它的视觉表达，而不是让普通单击变复杂。

## 当前实现对齐结论

当前图谱已有一些旧交互，其中一部分可以保留，一部分需要在后续实现时改语义。

保留：

- 滚轮/触控板在图谱区域内缩放图谱。
- 拖动空白处平移图谱。
- 工具栏里的“回全图”作为返回全图主入口。
- 双击空白回全图，作为辅助快捷方式。
- 节点优先于社区命中。

需要调整：

- 社区色块和社区图例的单击不应直接进入社区聚焦，应先显示社区摘要。
- 单击空白不应退出社区聚焦，只应取消当前选中。
- 节点单击在全局视角不应直接打开完整阅读，应先打开轻量节点摘要。
- 节点双击取消固定位置不应继续作为主设计，应改成明确按钮或菜单动作。
- 边的双击不应承担回全图，因为边未来可能需要轻量关系查看。

需要区分命名：

- “回全图”是视角导航：回到全局地图并 fit 全图。
- “重置布局”是布局操作：清除用户拖动形成的固定位置。
- 这两个动作不应共用同一文案或图标暗示。

## 旧交互调整方式

这一节用于把当前已有交互和新设计对齐。后续实现时，应按用户心智调整，不为了迁就旧行为保留隐蔽或多义的手势。

### 从“单击直接进入”改成“先检查，再深入”

社区色块和社区图例：

- 旧行为：单击后直接进入社区聚焦。
- 新行为：单击后只选中社区，右侧抽屉显示社区摘要，全局图谱保持原视角。
- 深入动作：用户在抽屉里点“进入社区”后，才进入社区聚焦。
- 辅助动作：“查看核心节点列表”只展开列表或高亮核心节点，不改变主图视角。

普通节点：

- 旧行为：单击后偏完整阅读。
- 新行为：单击后只显示节点轻量摘要，全局图谱保持原视角。
- 深入动作：用户在抽屉里点“进入所属社区”或“打开详情 / 阅读”。
- 不建议把节点双击作为进入社区或打开详情的主入口。

关系边：

- 旧行为：边双击可能承担回全图。
- 新行为：边不承担导航动作。
- 后续方向：边单击可以显示关系摘要，例如关系强度、两端节点、所属社区和相关证据。

### 从“空白处改变模式”改成“空白处只清选择”

空白单击：

- 在全局视角，只清空当前选中的节点或社区。
- 在社区聚焦里，只清空当前选中的节点或关系，仍留在当前社区。
- 不退出社区聚焦，不返回全图，不改变缩放。

空白双击：

- 只在真正点到图谱空白背景时，作为回全图快捷方式。
- 不在节点、社区色块、边、图例、搜索框、工具栏或右侧抽屉上触发。
- 它是快捷方式，不是主路径；主路径仍然是可见的“回全图”按钮。

### 从“隐藏手势”改成“明确按钮或菜单”

固定 / 取消固定：

- 旧行为：节点双击取消固定位置。
- 新行为：在节点轻量抽屉或节点工具菜单里提供“固定位置 / 取消固定位置”。
- 旧的双击取消固定最多作为过渡兼容，不在界面文案中强化，也不作为新用户必须学习的动作。

回全图：

- 使用明确的“回全图”按钮。
- 行为是回到全局地图并重新 fit 全图。
- 不清除 Pin，不清除用户固定的位置。

重置布局：

- 使用明确的“重置布局”按钮。
- 行为是清除用户拖动形成的固定位置，让布局重新整理。
- 不承担返回全图或退出社区聚焦。

### 抽屉内容随选中对象变化

全局视角共用一个右侧轻量抽屉，但内容和动作必须跟随选中对象变化：

- 选中社区：显示社区摘要、核心节点、桥接关系、“进入社区”、“查看核心节点列表”。
- 选中节点：显示节点摘要、所属社区、重要邻居、强关系、“进入所属社区”、“打开详情 / 阅读”。
- 没有选中对象：显示全局概览、搜索结果或空状态。

这能避免用户误以为“点社区”和“点节点”进入的是同一个详情逻辑。抽屉可以是同一个位置、同一个组件，但用户看到的内容层级和动作必须不同。

## 抽屉设计

右侧抽屉分两个使用场景，不必一定是两套组件，但用户感知要清楚：

- 全局轻量抽屉：在全局地图里检查当前选中的节点、社区、搜索结果或全局概览。
- 社区阅读抽屉：进入社区聚焦后承接更完整的阅读和理解。

### 全局轻量抽屉

目的：检查当前选中的对象，不打断地图。

选中节点时的推荐结构：

1. 标题区：节点名、类型、所属社区、连接数。
2. 关系区：最强关系 1-2 条。
3. 邻居区：最重要相邻节点 3-5 个。
4. 动作区：进入所属社区、打开详情。
5. 扩展区：后续可接入询问 Agent，但不影响第一轮抽屉设计。

选中社区时的推荐结构：

1. 标题区：社区名、节点数量、核心节点数量。
2. 摘要区：社区主题和主要内容范围。
3. 结构区：核心节点、桥接到其他社区的关系。
4. 动作区：进入社区、查看核心节点列表。

视觉要求：

- 窄、轻、信息克制。
- 不使用大段正文。
- 不撑大图谱布局。
- 同一个抽屉根据选中对象换内容，不额外弹第二层面板。

### 社区阅读抽屉

目的：帮助理解一个社区。

推荐结构：

1. 社区标题和摘要。
2. 核心节点列表。
3. 内部强关系。
4. 桥接到其他社区的关系。
5. 可进一步展开一跳邻居或打开节点详情。
6. 后续可接入向 Agent 发问，但不影响第一轮抽屉设计。

视觉要求：

- 信息可以比全局轻量抽屉更重。
- 允许阅读型内容。
- 仍避免一次塞满所有节点。

## 验收标准

设计层验收：

1. 用户能清楚理解：全局是地图，社区是阅读。
2. 普通节点点击不会让用户突然丢失全局上下文。
3. 社区色块/图例点击作为社区检查入口合理保留，进入社区聚焦需要再点明确按钮。
4. 右侧抽屉不再被定义为完整阅读入口，而是根据状态承担轻量查看或社区阅读。
5. 5000+ / 10000+ 大图有明确降级规则。
6. 方案没有同时引入多套全局图谱渲染主线。
7. 选中优先级清楚：节点 > 社区 > 搜索列表 > 全局概览。
8. 社区聚合被定义为全局渲染层级，而不是额外模式。
9. 返回全图有可见按钮，双击空白只是辅助快捷方式。
10. 单击空白不承担返回全图或退出社区聚焦。

后续实现验收：

1. 1000 节点只作为回归基线，不作为目标上限。
2. 1000 节点全局缩放、平移、悬停、节点点击保持流畅。
3. 5000+ 节点全局视角能稳定浏览。
4. 10000+ 节点至少能通过社区聚合或强降级视图流畅浏览。
5. 全局节点点击只打开轻量抽屉，不自动进入社区。
6. 社区色块/图例点击先显示社区摘要，点击“进入社区”后进入社区聚焦。
7. 社区聚焦后卡片和阅读信息恢复，但受核心节点数量限制。
8. 交互过程中弱边、普通标签和复杂效果可以临时隐藏。
9. 全局图谱不能因为抽屉内容变化而重排大图。
10. 全局中点击空白处能清空当前选中对象，抽屉回到全局概览或搜索结果列表。
11. 搜索、筛选、Pin 不应自动覆盖用户当前选中的节点或社区。
12. 社区聚焦中单击空白不返回全图；点“回全图”或双击空白才返回全图。
13. 节点双击不进入社区，不返回全图；固定/取消固定应有明确按钮或菜单入口。
14. “回全图”和“重置布局”在文案和行为上保持区分。

## 后续实现建议

下一步不应直接重写渲染器。

建议先做三件事：

1. 固化性能测量：覆盖 1000、5000、10000 节点的全局浏览和社区聚焦。
2. 定义降级策略：节点、边、标签、社区聚合分别在什么规模下触发。
3. 选型试验：用同一份图谱数据分别验证 `Sigma/Graphology` 与 `vis-network` 的全局大图表现，只保留一个全局渲染主线。

通过试验后再写实现计划。
