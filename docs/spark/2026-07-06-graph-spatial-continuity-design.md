# 图谱空间连续性推进设计

日期：2026-07-06
状态：已与用户确认第一批范围，待实现
当前分支：`codex/design-graph-spatial-continuity`
后续事项：社区图谱骨架视觉另见 GitHub issue #116

## 目的

这份设计用于收敛 llm-wiki 图谱里的空间过渡体验。

当前图谱已经分成两层：全局图谱回答“知识库整体长什么样”，社区图谱回答“这一组内容内部是什么关系”。现有 Sigma 路线已经有社区高亮、社区阅读、节点选择、Shift 多选和右侧抽屉联动，但几个关键动作仍有硬切感：

- 全局社区高亮返回全图时，视觉直接跳回全局。
- 从全局社区摘要进入社区图谱时，虽然有动效，但还不像同一张地图继续钻进去。
- 从社区图谱返回全图时，视觉是硬切。
- 社区图谱里打开或关闭右侧节点详情抽屉时，画布让位和恢复不够连续。
- 社区图谱里单击节点和 Shift 多选的视觉语言需要明确区分。

本设计目标不是为了“加动画”，而是让用户形成稳定心智：

**全局图谱和社区图谱是同一张地图的远近两层，右侧抽屉是这张地图旁边的阅读面板。**

## 已确认事实

本设计基于当前代码和产品事实，不重新定义已经落地的规则。

- 社区图谱回全图后保留来源社区高亮，这已经是现有设定，本次不改变。
- 全局图谱点社区会进入全局社区高亮态，并有相机聚焦能力。
- 点击“进入社区”后，右侧社区摘要抽屉已经会自动关闭。
- 社区图谱下 Shift 多选第一次会打开选区抽屉，继续多选时抽屉实时更新。
- 社区图谱已有标签预算：放大后更多标签出现，缩小后标签减少。
- 社区图谱已有节点大小、重要度、核心/相关/边缘等基础分层，但默认骨架可读性仍需后续单独设计。

## 工程评审后收紧点

以下是 2026-07-06 工程评审后对执行边界的补充，和 GitHub #118、#122、#123 保持一致：

- 第一张实现票不只是补“回全图动画”，还要建立后续空间连续性共用的 Sigma 视图过渡基座。#120、#121、#122 应复用同一套相机过渡、用户打断、减少动态效果和完成清理规则。

## 设计原则

### 1. 空间连续性优先

第一批只解决“我在哪里、我从哪里来、我正看哪里”的连续性。

用户进入社区时，应感觉自己从全局地图钻进一片区域；返回全图时，应感觉自己从局部退回整体，而不是被切到另一张图。

### 2. 用户操作优先

图谱是可操作地图，不是播放动画。

如果镜头正在移动，用户滚轮缩放、拖动画布、点击别的节点或点击别的社区，当前动效应停止或让位，优先响应新的用户意图。

### 3. 单击阅读，多选攒材料

社区图谱里：

- 单击节点表示“我要读这个节点”。
- Shift 多选表示“我要临时攒一组选区”。

两者动效必须不同。单击节点可以移动镜头给抽屉让位；Shift 多选不移动镜头，只更新选区反馈和抽屉内容。

### 4. 抽屉是阅读面板，不是新页面

右侧抽屉打开或关闭时，图谱不应硬切变窄或变宽。画布尺寸变化、抽屉进出和相机让位应该形成一段连续动作。

关闭抽屉只是收起详情，不自动把社区图谱镜头拉回中心。

### 5. 尊重减少动态效果

如果用户系统设置减少动态效果，保留状态变化，但取消大幅移动、分层浮现和复杂过渡。

## 第一批范围

第一批只做四条主路径。

### 1. 全局社区高亮返回全图

当前问题：从全局社区高亮态返回全图时，视觉硬切。

期望体验：

- 用户点“回全图”或等价返回入口。
- 镜头带着当前社区高亮拉远到全图构图。
- 镜头到位后，当前社区高亮再淡出。
- 其他社区和节点回到普通全局状态。

这里的关键是：高亮不要一开始就消失。它要陪着镜头拉远，让用户看清刚才看的区域在全局里的位置。

### 2. 全局进入社区

当前问题：点“进入社区”后，右侧社区摘要抽屉已经会关闭，但抽屉退场、画布扩展、镜头推进和社区内部结构呈现没有形成同一段体验。

期望体验：

- 用户在社区摘要抽屉里点“进入社区”。
- 抽屉立刻开始退场。
- 画布随抽屉退场平滑扩展。
- 镜头从全局社区高亮近景继续推进到社区阅读近景。
- 其他社区淡出。
- 当前社区内部关系浮现。

动效语言：全图到社区是一整个视觉衔接，而不是两个视图硬切。

进入社区可以采用短导览节奏：

1. 先保留社区轮廓和核心节点。
2. 再让骨架关系出现。
3. 最后补上浅背景关系。

这个导览不应拖沓，总时长控制在半秒左右。

### 3. 社区返回全图

当前问题：从社区图谱回全图时是硬切。

期望体验：

- 用户点社区图谱里的“回全图”。
- 社区细节快速退下去。
- 骨架关系短暂保留。
- 镜头拉回全图。
- 其他社区回到画面。
- 回到全局后保留来源社区高亮。

退出社区不是重点，用户已经决定离开局部阅读，所以这里要更短、更干净，不做复杂表演。

### 4. 社区单击节点和 Shift 多选

当前问题：单击节点打开右侧详情抽屉时，社区图谱给抽屉让位是硬切；单击阅读和 Shift 多选的动效边界需要固定。

期望体验：

- 单击节点后，节点和一阶关系先即时反馈。
- 右侧抽屉滑入。
- 画布平滑变窄。
- 镜头把被选节点带到剩余画布的舒服位置。
- 默认只移动镜头，不改变缩放比例；只有节点太靠边或关系圈被裁切时，才轻微缩小。
- 抽屉标题区域和图上节点做轻微视觉呼应。
- 关闭抽屉时，图谱宽度平滑恢复，但镜头不自动回社区中心。

Shift 多选：

- 第一次 Shift 多选打开选区抽屉。
- 后续继续 Shift 选节点，抽屉安静实时更新。
- 不重开抽屉，不闪烁，不移动镜头。
- 被选节点清晰，选中节点之间真实存在的关系线变亮。
- 没有真实关系时，不凭空画关系线。
- 选区节点分散时，可以出现很轻的临时选区轮廓。它表达“当前临时选区范围”，不表达知识关系，也不等同社区。
- 选区轮廓可以跨越没有直接关系的节点，但视觉上必须像柔和范围，不能像线段连接。
- 继续 Shift 多选时，节点即时反馈；选区轮廓稍慢、柔和地跟随变形。

## 附带小修

选区抽屉里的“选中页面”目前默认只显示 3 个节点。全局社区简介抽屉里的“核心节点”已有“查看全部 / 收起”模式，选区抽屉也应补上同样能力：

- 默认显示前 3 个选中页面。
- 超过 3 个时显示“查看全部”。
- 展开后显示全部页面，并显示“收起”。
- 继续 Shift 多选时保持用户当前展开或收起状态，不因为列表更新自动重置。

这属于多选路径的小修，不改变第一批主目标。

## 暂不做范围

以下内容不进入第一批，避免把空间连续性做散：

- 社区图谱骨架线选择、数量上限、颜色和主次规则，已记录到 GitHub issue #116。
- 社区图谱默认关系主次的完整视觉重做。
- 节点大小、标签和画布环境的进一步自适应重做。
- 新增一套关系类型说明或重做图例。
- 改变社区、选区、节点、关系这些领域语义。
- 改变知识库图谱数据生成逻辑。

## 动效节奏

统一三档节奏，避免每个入口各写一套手感：

### 即时反馈

目标：让用户知道点中了。

适用：

- 节点选中。
- 关系线变亮。
- 多选节点增删。
- 抽屉标题和节点轻微呼应。

建议时长：约 100ms。

### 布局、抽屉、镜头过渡

目标：让画布、抽屉和镜头形成同一段空间动作。

适用：

- 全局社区高亮返回全图。
- 全局进入社区。
- 社区返回全图。
- 社区单击节点后抽屉让位。
- 抽屉关闭后画布恢复。

建议时长：250-450ms。

### 导览分层

目标：进入社区时帮助用户理解层级。

适用：

- 进入社区时的“轮廓 -> 骨架 -> 细节”短导览。
- 分散多选时的选区轮廓柔和变形。

建议时长：500-650ms。退出社区时应更短。

## 验收标准

### 全局社区高亮返回全图

- 从全局高亮态返回全图时，不再硬切。
- 镜头先拉回全图，高亮随后淡出。
- 动效期间用户滚轮、拖拽或点击其他对象，可以打断当前过渡。
- 减少动态效果下，状态正确变化，但不做大幅移动。

### 全局进入社区

- 点击“进入社区”后，抽屉关闭、画布扩展、镜头推进和社区内部结构出现是连续的。
- 过渡后仍在 Sigma 社区阅读主路径。
- 进入社区后不自动重新打开社区摘要抽屉。
- 用户能感到这是同一张地图的近景，不是切到另一张图。

### 社区返回全图

- 点击“回全图”后不硬切。
- 社区细节退下，镜头回到全局。
- 返回全局后保留来源社区高亮。
- 退出过程简短，不做复杂导览。

### 社区单击节点

- 单击节点后，节点和一阶关系先反馈。
- 抽屉打开、画布让位、镜头移动同步发生。
- 被选节点进入剩余画布的舒服位置。
- 默认不改变缩放，必要时只轻微缩小防裁切。
- 关闭抽屉时画布平滑恢复，镜头不自动回社区中心。

### Shift 多选

- 第一次 Shift 多选打开选区抽屉。
- 后续 Shift 多选不移动镜头。
- 抽屉安静实时更新，不重开、不闪。
- 选中页面超过 3 个时有“查看全部 / 收起”。
- 真实存在的选区内关系可被强调，不画不存在的关系。
- 临时选区轮廓不会被误认为社区或关系线。

## 推进顺序

建议实现顺序：

1. 补齐镜头返回全图的动画能力，让全局高亮态退出不硬切。
2. 打通全局进入社区和社区返回全图的过渡状态。
3. 做抽屉打开/关闭时的画布尺寸连续和节点让位。
4. 固定 Shift 多选不移动镜头，并补选区抽屉“查看全部 / 收起”。
5. 做浏览器实测：普通屏、外接大屏、抽屉打开、抽屉关闭、减少动态效果、用户中途打断。

每一步都要保持现有社区回全图后来源社区高亮的设定。
