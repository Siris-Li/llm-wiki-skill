# 一个图谱引擎，两个宿主

> 状态：repo-wide 摘要。图谱引擎的详细设计、交互修订和重评估条件以 ADR-21 为准；本 ADR 只记录领域文档拆分后的跨入口约束。

工作台图谱视图和 Skill 离线 HTML 共用 `packages/graph-engine`。同一套图谱语义和交互核心产出两种形式：ESM 给工作台使用，IIFE 给离线 HTML 打包；宿主差异通过能力开关表达，不分叉两套引擎。

**备选方案**

- 工作台和离线 HTML 各写一套图谱：短期更自由，但会很快出现行为漂移和双份维护。
- 工作台直接嵌入离线 HTML：复用更快，但图谱会变成孤岛，难以和对话、抽屉、选区联动。
- 只服务工作台，不升级离线 HTML：开发更轻，但 Skill 用户拿不到图谱体验升级。

**影响**

- 图谱语义、选择、社区、钉扎和阅读动作优先落在 `packages/graph-engine`。
- 工作台专属的提问、抽屉和应用状态通过宿主能力接入，不写死进离线 HTML。
- 修改图谱行为时，必须同时考虑工作台视图和 Skill 离线 HTML。
- 如果本 ADR 与 ADR-21 的细节不一致，按 ADR-21 执行，再同步更新本摘要。
