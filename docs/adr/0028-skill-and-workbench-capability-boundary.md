# Skill 与工作台的能力边界

已经在 Skill 里成熟存在的知识库主线能力继续归 Skill 维护，包括初始化、消化、查询、深度综合、健康检查、状态、图谱、删除和手动结晶（`crystallize`：整理用户主动提供的内容）。只属于工作台的产品能力，例如当前知识库状态、页面列表、对话管理、当前对话结晶、产出物和图谱交互状态，放在工作台侧，不塞回 Skill。

“结晶”这个词在两个入口里必须带限定词：Skill 的手动结晶不读取工作台当前会话；工作台的对话结晶可以读取当前对话和界面状态，并调用写入 wiki 页面的能力。这个边界延续 ADR-16 的能力归属原则。

**备选方案**

- 在工作台里重写 Skill 工作流：看起来更一体化，但会复制成熟逻辑，造成两套事实来源。
- 把工作台专属功能塞进 Skill：会降低 Skill 的可移植性，也会把应用状态和提示词工作流混在一起。

**影响**

- 已经存在为 Skill 工作流的能力，优先调用或复用。
- 依赖工作台界面状态、对话、产出物或当前选区的能力，留在工作台侧。
- 提到结晶时必须说明是 Skill 手动结晶还是工作台对话结晶。
- 例外必须明确、范围要窄。
