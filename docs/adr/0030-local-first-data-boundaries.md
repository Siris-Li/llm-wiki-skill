# 本地优先与数据边界

llm-wiki 保持本地优先：知识库放在用户自己的本地目录，工作台应用状态放在应用数据目录，模型凭证交给 pi-agent 的凭证体系。这个项目不应变成云服务，不应把 API key 写进自己的配置，也不应把知识内容和应用偏好混在一起。

“本地优先”不表示模型调用期间没有外部信任边界。用户 prompt、选中的引用、检索片段、工具输出和生成产物可能发送给当前配置的模型提供商；工作台必须按最小必要原则组织上下文，不能暗示知识内容在模型调用时永远不离开本机。

**备选方案**

- 云端优先服务：上手和同步更容易，但会改变产品方向、成本结构、隐私边界和维护负担。
- 所有东西放进同一个应用目录：更好找，但会降低知识库可迁移性，也增加凭证风险。

**影响**

- 知识内容、应用状态和模型凭证必须分开。
- 工作台能力不应依赖云端托管。
- 新增存储位置时，必须说明它属于哪条边界。
- 模型提供商是外部信任边界；新增自动上下文注入时，必须说明会发送哪些内容。
