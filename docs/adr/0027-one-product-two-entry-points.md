# 一个产品，两种入口

llm-wiki 是一个产品，核心是一套本地知识库格式、素材管线和知识构建方法。Skill 形态和 agent 工作台都保留：Skill 让用户把 llm-wiki 带进已有 AI 工具，工作台则提供专门的本地应用，用来承载对话、图谱、产出物和多模型流程。

**备选方案**

- 只保留 Skill：维护更简单，但做不出完整工作台体验。
- 只保留工作台：产品形态更单一，但会放弃已有用户，也削弱“知识库格式不绑定工具”的承诺。

**影响**

- 共用概念写在根目录 `CONTEXT.md`。
- 入口专属概念写在各自的局部 `CONTEXT.md`。
- 改知识库格式时，必须同时考虑 Skill 用户和工作台用户。
