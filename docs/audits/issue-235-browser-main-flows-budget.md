# #235 browser-main-flows 作业预算证据

## 完成与验证标准

- 依赖、系统准备、浏览器下载与安装、浏览器启动验证、正式测试和诊断上传各自可观察，并有独立失败边界。
- 冷缓存会完整准备环境；缓存命中只省略浏览器下载与安装，不能省略系统依赖或浏览器启动验证。
- 安装失败或不退出时，相关进程在阶段时限内结束，退出状态和脱敏输出进入诊断材料。
- 正式测试失败后，runner 完成清理并生成脱敏证据，作业仍有时间执行上传。
- 不增加固定休眠、自动重试、选择性重跑、跳过正式测试或放宽产品断言。
- PR 最新提交的 `quality-and-tests` 与 `browser-main-flows` 必须真实成功；合并后的 `main` 复验留给合并收尾。

## 阶段与缓存边界

| 阶段 | 发生位置 | 内部上限 | 工作流上限 | 可观察证据 |
| --- | --- | ---: | ---: | --- |
| npm 依赖 | `npm ci` | 90 秒 | 2 分钟 | 阶段 JSON、脱敏日志、退出码 |
| 缓存身份 | 锁文件、Playwright 包版本、浏览器清单 | 不适用 | 1 分钟 | 身份 JSON、步骤输出 |
| 缓存恢复 | `actions/cache` | 不适用 | 2 分钟 | GitHub 步骤结论与 `cache-hit` |
| 系统依赖 | `playwright install-deps chromium` | 8 分钟 | 9 分钟 | 阶段 JSON、脱敏日志、退出码 |
| 浏览器下载与安装 | `playwright install chromium`，仅缓存未命中 | 3 分钟 | 4 分钟 | 每个内容的下载开始、落盘完成时间点及阶段结果 |
| 浏览器验证 | 实际启动并关闭 Chromium | 45 秒 | 1 分钟 | 阶段 JSON、启动结果 |
| 正式测试 | 既有 browser-main-flows runner | 270 秒 | 5 分钟 | runner 输出、阶段 JSON、退出码 |
| 诊断上传 | `actions/upload-artifact` | 不适用 | 2 分钟 | GitHub 步骤与 artifact |

工作流步骤上限合计 28 分钟，作业上限 32 分钟，保留 4 分钟给步骤切换、runner 清理、Action 收尾和诊断处理。系统依赖的 8 分钟内部边界来自历史 7 分 51 秒停顿；它会在作业总时限前明确失败。浏览器安装和正式测试上限分别以成功运行约 9 秒下载与安装、最长 45 秒正式测试为基线，并保留数倍余量。此次本地代表性实测为：`npm ci` 8.873 秒，浏览器启动验证 0.821 秒，正式测试阶段 38.275 秒。

缓存目录为工作区内的 `.cache/ms-playwright`。精确键由 `runner.os`、`runner.arch`、锁文件中的 Playwright 版本，以及 Playwright 清单内 Chromium、headless shell 和 FFmpeg 的版本与 revision 共同决定；没有宽松恢复键。系统或架构变化、Playwright 锁定版本变化、实际浏览器内容变化都会未命中。缓存未命中执行完整安装；命中只跳过该安装，系统依赖与实际启动验证仍执行。

## 失败与诊断路径

| 场景 | 预期观察 |
| --- | --- |
| 冷缓存 | cache miss，浏览器下载与安装步骤运行并记录内容时间点，随后浏览器验证和正式测试成功 |
| 常规运行 | 精确 cache hit，浏览器安装步骤跳过，系统依赖、浏览器验证和正式测试仍成功 |
| 系统依赖失败 | 系统依赖步骤返回非零，记录退出码和脱敏日志，正式测试不启动，诊断上传执行 |
| 浏览器安装失败 | 浏览器安装步骤返回非零，记录已到达的下载时间点、退出码和脱敏日志，诊断上传执行 |
| 安装不退出 | 内部时限触发，子进程树被清理，状态为 `timed-out`，诊断上传执行 |
| 正式测试失败 | 既有 runner 先清理浏览器、前后台与端口并写入脱敏证据；外层阶段记录失败，上传同时包含 runner 与阶段证据 |
| 作业被整体取消 | GitHub 不再调度后续上传；32 分钟上限大于所有步骤上限之和，正常阶段失败不会先耗尽作业总时限 |

直接负向验证使用受控短时子进程和退出事件，不使用固定休眠，也不删除或覆盖任何 GitHub 缓存。runner 原有失败、中断、超时清理与脱敏路径继续由共享进程执行器和新增外部 runner 验证覆盖。
